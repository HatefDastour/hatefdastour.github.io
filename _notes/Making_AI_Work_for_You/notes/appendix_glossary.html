Appendix: Glossary of Key Terms

7. Appendix: Glossary of Key Terms#

Agents

Autonomous workflows capable of multi-step planning, tool use (such as code execution or APIs), and self-correction to achieve a complex goal.

AIM Framework

A structured approach to prompt engineering consisting of Actor (or Audience), Instruction (or Intent), and Methodology. The “A” can represent either the Actor (the role the model should assume, such as “expert tutor”) or the Audience (the intended reader of the output). Similarly, while “Intent” is common for the “I,” many versions use “Instruction” to emphasize the specific command or task given to the model.

CARE Framework

A specialized prompt framework standing for Context, Action, Result, and Example; it is best utilized for synthesizing information.

Chain-of-Thought (CoT)

A prompting technique that encourages step-by-step reasoning by asking the model to show intermediate steps, which helps reduce errors and hallucinations.

Context Window

The maximum amount of text (measured in tokens) that an LLM can process in a single interaction. Information outside this window is not “seen” by the model.

CREATE Framework

A comprehensive framework standing for Character, Request, Examples, Additions, Type, and Extras; it is ideal for complex writing and drafting.

Embedding

High-dimensional vector representations of text that capture semantic meaning, allowing the model to understand relationships between words and concepts in a mathematical space.

ERA Framework

A foundational prompt framework standing for Expectation, Role, and Action; it is designed for completing quick, straightforward tasks.

Few-Shot Learning

Providing 1–5 specific examples of desired input-output behavior within a prompt to teach the model patterns, style, or domain-specific formatting.

Hallucination

When an LLM generates false or fabricated information that sounds plausible but is factually inaccurate, such as inventing citations or technical facts.

In-Context Learning

The ability of LLMs to learn and perform tasks based solely on examples and instructions provided in the prompt, without changing the model’s underlying parameters.

OCEAN Framework

A quality-focused framework standing for Objective, Context, Examples, Action, and Needs; it is used to sharpen output quality and ensure all requirements are met.

Parameter

Numerical values that represent the model’s capacity and reasoning power; generally, more parameters enable more complex reasoning.

Prompt Engineering

The art and science of crafting effective instructions and queries to maximize the quality, relevance, and accuracy of LLM outputs.

Quantization

A compression technique that reduces the precision of model parameters to make models smaller and faster with minimal loss in quality.

RAG (Retrieval-Augmented Generation)

A workflow where the LLM searches specific verified documents (like your own PDFs) to provide cited, grounded answers, ensuring the model uses information beyond its original training data.

Recall Decay

A technical limitation where models struggle to find or use details that are buried in the middle of a very long text, even if that text fits within the context window.

ROSES Framework

A strategic framework standing for Role, Objective, Scenario, Expected Solution, and Steps; it is best suited for strategic planning.

Semantic Search

A search method that uses embeddings to find relevant information based on the conceptual meaning of a query rather than just matching exact keywords.

Show-Me AI

Mizzou’s secure generative AI platform that is DCL-3 compliant, ensuring that sensitive research and teaching data stays private on campus servers. “DCL-3” refers to the University of Missouri’s Data Classification Level 3, which designates restricted data requiring enhanced protection.

Temperature

A basic model setting that influences the randomness or creativity of the generated text. Lower values produce more deterministic and factual outputs, while higher values increase randomness and creative diversity in responses.

Token

The basic unit of text (roughly 0.75 words) that LLMs process. Context limits, processing speed, and costs are all measured in tokens.

Zero-Shot Learning

Asking the model to perform a task without providing any examples, relying entirely on the provided instructions and its pre-trained knowledge.
