1.4.2. Morphology and Part-of-Speech Tagging#

1.4.2.1. What is Morphology?#

Morphology is the branch of linguistics that studies how words are formed from smaller meaningful units called morphemes.

Key Term — Morpheme

A morpheme is the smallest meaningful unit of language. For example, "unhappiness" consists of three morphemes: un- (negation prefix) + happy (root) + -ness (noun-forming suffix). Morphemes are not the same as syllables — "unhappiness" has four syllables but only three morphemes.

There are two major types of morphology relevant to NLP:

  • Inflectional morphology: changes grammatical features (tense, number, gender) without altering the core meaning of the word.
    Examples: run → runs → running, child → children, big → bigger → biggest.

  • Derivational morphology: creates new words or changes the word class (part of speech).
    Examples: govern → government → governmental, happy → happiness → unhappy.

Understanding morphology helps explain why lemmatization and stemming (Section 1.3) work the way they do: we are trying to map inflected forms back to a common base, which is a morphological operation.

1.4.2.2. Part-of-Speech (POS) Tagging#

In NLP, we often approximate morphological information through part-of-speech (POS) tags, which label each token with a coarse grammatical category (noun, verb, adjective, etc.).

POS tagging serves several purposes:

  • Provides a compact summary of sentence structure.

  • Supports downstream tasks such as parsing, information extraction, and named entity recognition.

  • Improves feature representations for classical models (e.g., “verb lemmas in questions”).

  • Disambiguates word senses: "saw" can be a past-tense verb or a tool noun — the POS tag tells us which.

Each token is associated with:

  • A coarse POS tag (token.pos_), such as NOUN, VERB, ADJ. These are universal across languages.

  • A fine-grained tag (token.tag_), such as NN (singular noun) or VBZ (3rd-person singular present verb). These are English-specific (Penn Treebank).

  • A dependency relation (token.dep_) indicating its syntactic role relative to a head word.

1.4.2.7. Example: Filtering by POS for Keyword Extraction#

A common applied pattern is to extract only content-bearing tokens (nouns, proper nouns, and main verbs) from a text. POS filtering produces a compact, informative summary of what a document is about:

Show code cell source

Hide code cell source

 
import spacy

nlp = spacy.load("en_core_web_sm")

text = (
    "The IPCC Sixth Assessment Report highlights that global temperatures have risen "
    "by approximately 1.1°C since pre-industrial times, driven primarily by human activities "
    "such as burning fossil fuels and deforestation."
)

doc = nlp(text)

# Extract nouns, proper nouns, and main verbs
content_words = [
    token.lemma_.lower()
    for token in doc
    if token.pos_ in ("NOUN", "PROPN", "VERB")
    and not token.is_stop
    and len(token.lemma_) > 2
]

print("Content words:", content_words)

Content words: ['ipcc', 'sixth', 'assessment', 'report', 'highlight', 'temperature', 'rise', 'time', 'drive', 'activity', 'burn', 'fossil', 'fuel', 'deforestation']

This is a simple, interpretable keyword extraction approach. It requires no training data and works immediately on any domain — useful as a rapid prototyping step before moving to more sophisticated methods.

1.4.3. Named Entity Recognition#

In addition to POS tags, spaCy provides Named Entity Recognition (NER): the identification of real-world objects such as persons, organizations, dates, and locations mentioned in text.

Key Term — Named Entity

A named entity is a real-world object that can be referred to with a proper noun: a person, an organization, a location, a date, a monetary amount, etc. NER is the task of finding all such spans in text and labeling each with its type.

1.4.4. Syntactic Structure#

Syntax concerns how words combine into phrases and sentences. Understanding syntactic structure is important for:

  • Question answering and information extraction (e.g., identifying who did what to whom).

  • Coreference resolution and discourse analysis.

  • Error analysis and interpretability of model behavior.

While large language models often learn to exploit syntactic regularities implicitly, explicit parses remain useful for many applications and research questions.

1.4.4.1. Dependency vs. Constituency (Intuition)#

Two classical representations of syntactic structure are used in NLP:

  • Constituency trees represent hierarchical phrase structure, grouping tokens into nested constituents: [NP The large language model] [VP [V summarizes] [NP scientific articles] [ADVP accurately]]. Each non-leaf node is a phrase category (NP = noun phrase, VP = verb phrase, etc.).

  • Dependency graphs represent head–modifier relationships as a directed graph. Each token points to its head (the word it depends on), with a labeled arc indicating the relationship type. The root of the sentence has no head.

1.4.4.4. Extracting Subject–Verb–Object Triples#

Dependency parses make it straightforward to extract who-does-what information, a technique sometimes called Open Information Extraction (OpenIE):

Show code cell source

Hide code cell source

 
import spacy

nlp = spacy.load("en_core_web_sm")

sentences = [
    "The flood destroyed several bridges in the valley.",
    "Scientists published new findings about sea level rise.",
    "The government announced a new climate adaptation policy.",
]

for sent in sentences:
    doc = nlp(sent)
    for token in doc:
        if token.dep_ == "ROOT":
            subj = [t.text for t in token.lefts  if t.dep_ in ("nsubj", "nsubjpass")]
            obj  = [t.text for t in token.rights if t.dep_ in ("dobj", "obj", "pobj")]
            print(f"Verb: {token.text:15s} | Subject: {subj} | Object: {obj}")

Verb: destroyed       | Subject: ['flood'] | Object: ['bridges']
Verb: published       | Subject: ['Scientists'] | Object: ['findings']
Verb: announced       | Subject: ['government'] | Object: ['policy']

This kind of structured extraction — transforming free text into (subject, verb, object) triples — is a building block for constructing knowledge bases and populating databases from unstructured text.

1.4.5. Semantic Representations#

Beyond syntax, we care about meaning. Classical NLP used several representation schemes to capture meaning in a form that statistical models can process.

1.4.5.1. Bag-of-Words and TF-IDF#

The bag-of-words (BoW) model represents a document as a vector of word counts, discarding word order. Each dimension corresponds to a vocabulary item.

Term Frequency–Inverse Document Frequency (TF-IDF) is a weighted version of BoW that de-emphasizes very common words and highlights words that are distinctive to a document:

TF-IDF Formula

For a term t in document d from a corpus of N documents:

(1.1)#\[\begin{equation} \text{TF-IDF}(t, d) = \underbrace{\text{count}(t, d)}_{\text{term frequency}} \times \underbrace{\log \frac{N}{\text{df}(t)}}_{\text{inverse document frequency}} \end{equation}\]

where df(t) is the number of documents that contain term t.

  • TF (term frequency): how often t appears in d. High for words important to this specific document.

  • IDF (inverse document frequency): the log ratio of total documents to documents containing t. High for rare words; low for common words like “the”.

The intuition: a word that appears 10 times in a document but also appears in 90% of all documents is not very informative. A word that appears 10 times but only in 2% of documents is likely a key topic term.

1.4.5.2. Static Word Embeddings#

Before contextual embeddings, static word embeddings such as word2vec and GloVe were the standard way to represent meaning. They map each word to a fixed dense vector of real numbers (typically 50–300 dimensions), where words with similar meanings are geometrically close.

How word2vec works (high level):

Word2vec trains a shallow neural network to predict either:

  • Skip-gram: given a word, predict its surrounding context words (within a window).

  • CBOW (Continuous Bag of Words): given the surrounding context words, predict the target word.

Neither task is the true goal — the goal is the learned weights (the embedding matrix), which encode co-occurrence patterns from the corpus. Words that appear in similar contexts end up with similar vectors.

The famous result king − man + woman ≈ queen illustrates the geometric structure of these embeddings:

Static embeddings capture broad semantic proximity, but they cannot distinguish “river bank” (financial institution) from “river bank” (natural feature) — that is the key limitation that contextual embeddings address.

1.4.5.3. The Polysemy Problem and Contextual Embeddings#

Static embeddings assign a single fixed vector to each word. This means the word “bank” has one vector regardless of whether it means a financial institution, a riverbank, or the act of banking a curve. This is the polysemy problem: one form, many meanings.

Contextual embeddings, produced by transformer-based models (BERT, RoBERTa, LLaMA, etc.), solve this by generating a different vector for each occurrence of a word based on its surrounding context. The representation of "bank" in “He deposited money at the bank” differs from its representation in “We sat on the river bank”.

This context-dependence is a fundamental architectural difference and one of the reasons transformers achieve much higher performance than static embedding approaches on most NLP benchmarks.

1.4.6. Historical Milestones in NLP#

It is useful to situate large language models within the broader history of NLP:

Table 1.26 Very high-level timeline of NLP methods#

Era

Dominant approach

Representative methods

Rule-based (1950s–1980s)

Hand-written grammars and rules

ELIZA, early MT systems

Statistical (1990s–2000s)

Probabilistic models over text and speech

N-grams, HMMs, CRFs, IBM MT models

Neural (2010–2016)

Large Language Models (2019–)

Massive pretrained models with in-context learning

Key transitions:

  • From rules to statistics: data-driven models replacing hand-crafted grammars. Statistical models were more robust and could generalize from examples, but required large annotated corpora.

  • From statistics to neural: learned dense representations reduced the need for manual feature engineering. RNNs could model sequences of arbitrary length, but struggled with long-range dependencies.

  • From RNNs to transformers: attention mechanisms enabled parallel training and eliminated the bottleneck of processing sequences step-by-step. Transformers can relate any two positions in a sequence directly, regardless of distance.

  • From transformers to LLMs: scaling data, parameters, and compute revealed emergent capabilities — tasks that small models could not do at all suddenly became possible with large models, without any task-specific training.

Throughout this evolution, linguistic concepts like morphology, syntax, and semantics have remained relevant. Even when models do not explicitly expose these structures, they often benefit from training data and evaluation protocols that reflect them.

1.4.7. Linguistic Structure and LLMs#

Although LLMs are not built as explicit linguistic pipelines, evidence from probing studies suggests that:

  • They encode substantial information about POS, syntax, and semantics in their internal representations.

  • They can often be probed for linguistic knowledge (e.g., predicting grammaticality, subject–verb agreement, or coreference).

  • They may still struggle with phenomena requiring explicit logical reasoning, deep world-knowledge chains, or careful discourse tracking.

Probing refers to a family of evaluation methods in which a small classifier is trained on the hidden states of a pretrained LLM to predict some linguistic property (e.g., POS tag, parse depth, semantic role). High probing accuracy indicates that the LLM’s internal representations encode that property.

In later chapters, we will revisit these ideas when we analyze how transformer-based models encode linguistic structure and how we can probe and interpret their internal representations.

Key Takeaways

  • Morphology and POS tagging provide a compact description of each token’s grammatical role. spaCy provides coarse universal tags (token.pos_) and fine-grained Penn Treebank tags (token.tag_) in a single pipeline call. Understanding tag sets helps you read and debug model output.

  • Named Entity Recognition (NER) identifies real-world objects in text (persons, organizations, dates, locations) and is the foundation for structured information extraction. NER performance is domain-dependent: general-purpose models may need re-training or prompting for specialized text.

  • Dependency parsing reveals head–modifier relationships, making it possible to extract who-does-what-to-whom (subject–verb–object triples) from raw text.

  • Constituency vs. dependency are two complementary views of syntax; most modern toolkits default to dependency because it is more compact and easier to integrate with downstream tasks.

  • TF-IDF weights term frequency by inverse document frequency, highlighting distinctive words and down-weighting common ones. The formula combines term-level and corpus-level statistics.

  • Static word embeddings (word2vec, GloVe) capture broad semantic similarity but assign a single vector per word type, unable to represent polysemy. Word2vec learns by training a shallow network on a co-occurrence prediction task.

  • Contextual embeddings from transformers solve polysemy by making representations context-dependent — the same word has different vectors in different sentences.

  • Understanding the history of NLP methods — from rules to statistics to neural networks to LLMs — clarifies why certain design decisions were made and when classical tools are still the right choice.

1.4.8. Exercises and Discussion#

  1. POS tagging intuition
    Take a short paragraph (3–4 sentences) from a news article. Without any tools, manually annotate POS tags for each word using the Universal Dependencies categories. Then run the paragraph through spaCy and compare results. Where do you disagree? Why might the model differ from your annotation?

  2. Dependency vs. constituency
    Sketch both a constituency tree and a dependency tree for the sentence:
    “The rapidly changing climate is affecting coastal communities worldwide.”
    Discuss what each view makes easy or hard to see.

  3. Subject–verb–object extraction
    Run the SVO extraction code above on three or four sentences from an environmental report or news article. Are the extracted triples accurate? What types of sentences (passive voice, subordinate clauses, coordination) cause errors?

  4. TF-IDF computation
    Given a corpus of five short sentences from a climate news article, compute TF-IDF by hand for the word “flood” in the sentence “The flood damaged crops”. Verify your calculation using TfidfVectorizer from scikit-learn.

  5. Historical reflection
    Choose one era from the timeline (e.g., “statistical NLP”). What were its strengths and limitations compared to LLM-based methods? In what situations might a classical method (e.g., a CRF-based NER tagger) still be preferable today?

  6. Static vs. contextual embeddings
    Consider the word “plant” in these two sentences:
    (a) “The power plant was built in 1980.”
    (b) “She watered the plant on her windowsill.”
    Would a static embedding handle these differently? How would a contextual embedding differ? What downstream tasks would break if both senses had the same vector?
