5.3.3.2. The Text Classification Problem#

Text classification is the task of assigning a label from a predefined set to a piece of text. It encompasses a wide range of applications:

The core challenge is the same across tasks: how do we convert variable-length text into a fixed-length feature vector that a classifier can operate on?

5.3.3.3. Three Generations of Approaches#

Text classification methods have evolved through three distinct generations, each with fundamentally different representations and models:

Generation 1 — Sparse, count-based features:
  TF-IDF → SVM or Logistic Regression
  TF-IDF → Naive Bayes
  Fast to train, highly interpretable, strong on clean balanced data.

Generation 2 — Dense, distributional embeddings:
  Word2Vec / GloVe + mean pooling → MLP classifier
  Word2Vec / GloVe + 1D CNN → CNN classifier
  Captures semantic similarity; handles synonyms; smaller vocabulary footprint.

Generation 3 — Contextualized transformer representations:
  BERT / RoBERTa fine-tuned end-to-end
  The same word receives different representations in different contexts.
  State-of-the-art performance; expensive to train.

The practices in this section implement all three generations on two benchmark datasets:

  • Rotten Tomatoes (8,530 training / 1,066 test): binary sentiment, short critic snippets.

  • Yelp Polarity (560K available, 12,000 used): binary sentiment, longer business reviews.

5.3.3.4. Generation 1: Classical Sparse Pipelines#

5.3.3.4.1. TF-IDF + SVM (Rotten Tomatoes)#

TF-IDF (Term Frequency–Inverse Document Frequency) [Salton and McGill, 1983] converts each document to a sparse vector in which each dimension corresponds to a vocabulary term, weighted by how characteristic that term is of the document:

(5.30)#\[\begin{equation} \text{TF-IDF}(t, d) = \text{TF}(t, d) \times \log\frac{N}{1 + \text{df}(t)} \end{equation}\]

Support Vector Machines (SVMs) [Cortes and Vapnik, 1995] with a linear kernel find the maximum-margin hyperplane separating the two classes in this high-dimensional sparse space.

from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.svm import LinearSVC
from sklearn.pipeline import Pipeline

pipeline_svm = Pipeline([
    ("tfidf", TfidfVectorizer(max_features=20000, ngram_range=(1, 2),
                               sublinear_tf=True)),
    ("clf",   LinearSVC(C=1.0, max_iter=2000)),
])

pipeline_svm.fit(X_train, y_train)
y_pred = pipeline_svm.predict(X_test)

Typical accuracy on Rotten Tomatoes: ~78–82%. The SVM’s advantages are speed (seconds to train), interpretability (the highest-weight n-grams reveal what the model has learned), and competitive performance on clean, balanced datasets.

5.3.3.4.2. TF-IDF + Naive Bayes (Yelp)#

Multinomial Naive Bayes [McCallum and Nigam, 1998] applies Bayes’ theorem with a feature-independence assumption: given the label, each word’s presence is assumed independent of all other words. Despite this unrealistic assumption, it works well on text:

(5.31)#\[\begin{equation} P(\text{label} \mid \text{document}) \propto P(\text{label}) \prod_{t \in \text{document}} P(t \mid \text{label}) \end{equation}\]

5.3.3.6. Generation 1: Classical Sparse Pipelines#

5.3.3.6.1. TF-IDF + SVM (Rotten Tomatoes)#

TF-IDF (Term Frequency–Inverse Document Frequency) [Salton and McGill, 1983] converts each document to a sparse vector in which each dimension corresponds to a vocabulary term, weighted by how characteristic that term is of the document:

(5.32)#\[\begin{equation} \text{TF-IDF}(t, d) = \text{TF}(t, d) \times \log\frac{N}{1 + \text{df}(t)} \end{equation}\]

where \(\text{TF}(t,d)\) is the count of term \(t\) in document \(d\), \(N\) is the total number of documents, and \(\text{df}(t)\) is the number of documents that contain \(t\). Terms that appear in many documents receive low IDF weight; terms that are rare across the corpus but frequent in a specific document receive high weight.

Support Vector Machines (SVMs) with a linear kernel find the maximum-margin hyperplane separating the two classes in this high-dimensional sparse space. The sublinear_tf=True option replaces raw term counts with \(1 + \log(\text{count})\), which compresses the influence of very frequent terms.

5.3.3.6.2. TF-IDF + Naive Bayes (Yelp)#

Multinomial Naive Bayes [McCallum and Nigam, 1998] applies Bayes’ theorem with a feature-independence assumption: given the class label, each word’s presence is assumed to be independent of all other words. Despite this unrealistic assumption, the model works well in practice because the independence errors tend to cancel across a large vocabulary.

(5.33)#\[\begin{equation} P(\text{label} \mid \text{document}) \propto P(\text{label}) \prod_{t \in \text{document}} P(t \mid \text{label}) \end{equation}\]

In log-space — which is how the classifier is actually computed to avoid numerical underflow — this product becomes a sum of log-probabilities, making Naive Bayes computationally equivalent to a linear classifier over log-count features.

Key Definition — Multinomial Naive Bayes

Multinomial Naive Bayes [McCallum and Nigam, 1998] estimates the probability of each class given a document by applying Bayes’ theorem:

(5.34)#\[\begin{equation} P(y \mid \mathbf{x}) \propto P(y) \prod_{t=1}^{T} P(x_t \mid y) \end{equation}\]

where \(y\) is the class label, \(\mathbf{x} = (x_1, \ldots, x_T)\) are the term counts in the document, and:

  • \(P(y)\) is the class prior (proportion of training documents in class \(y\))

  • \(P(x_t \mid y)\) is the smoothed term likelihood, estimated with Laplace (add-\(\alpha\)) smoothing:

(5.35)#\[\begin{equation} P(t \mid y) = \frac{\text{count}(t, y) + \alpha}{\sum_{t'} \text{count}(t', y) + \alpha \cdot |V|} \end{equation}\]

The alpha=0.1 parameter in the code below controls this smoothing strength.

5.3.3.7. Generation 2: Dense Embedding Networks#

5.3.3.7.1. GloVe + Mean-Pooled MLP (Rotten Tomatoes)#

GloVe (Global Vectors) pre-trains word embeddings on a large corpus by factorizing a word co-occurrence matrix. Each word in the vocabulary receives a fixed 100- or 300-dimensional vector.

To classify a document, we:

  1. Look up the GloVe embedding for each word in the document.

  2. Mean pool all word vectors to produce a single fixed-length document vector.

  3. Pass this vector to a multi-layer perceptron (MLP) classifier.

5.3.3.7.2. GloVe + Mean-Pooled MLP (Rotten Tomatoes)#

GloVe (Global Vectors) [Pennington et al., 2014] pre-trains word embeddings by factorizing a word co-occurrence matrix derived from a large corpus. Each word receives a fixed 100- or 300-dimensional dense vector that encodes distributional similarity: words that appear in similar contexts are placed nearby in embedding space. Unlike TF-IDF, GloVe embeddings are shared across documents and are not re-estimated from the training set.

To produce a fixed-length document representation from a variable-length sequence of word vectors, we apply mean pooling: all word embeddings in the document are averaged element-wise. The resulting vector is passed through a multi-layer perceptron (MLP) to produce class logits.

The code below downloads the 100-dimensional GloVe vectors (822 MB compressed), constructs document embeddings for both datasets, and trains a two-layer MLP. The GloVe file is downloaded once and cached locally.

import numpy as np
import torch
import torch.nn as nn


def embed_documents(texts, glove, dim=100):
    embeddings = []
    for text in texts:
        vecs = [glove[w] for w in text.lower().split() if w in glove]
        if vecs:
            embeddings.append(np.mean(vecs, axis=0))
        else:
            embeddings.append(np.zeros(dim))
    return np.array(embeddings)


class MLPClassifier(nn.Module):
    def __init__(self, input_dim=100, hidden_dim=128, num_classes=2):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(input_dim, hidden_dim),
            nn.ReLU(),
            nn.Dropout(0.3),
            nn.Linear(hidden_dim, num_classes),
        )

    def forward(self, x):
        return self.net(x)

Key Definition — Mean Pooling

Mean pooling aggregates a variable-length sequence of word embeddings into a single fixed-length document vector:

(5.36)#\[\begin{equation} \mathbf{d} = \frac{1}{|\mathcal{W}|} \sum_{w \in \mathcal{W}} \mathbf{e}_w \end{equation}\]

where \(\mathcal{W}\) is the set of words in the document that appear in the vocabulary and \(\mathbf{e}_w \in \mathbb{R}^d\) is the pre-trained embedding for word \(w\). Mean pooling is order-invariant: shuffling the words in a document yields the same vector. This is both a strength (no sensitivity to minor word-order variations) and a limitation (negation and word order are invisible).

def coverage(texts, glove):
    total, found = 0, 0
    for t in texts:
        for w in t.lower().split():
            total += 1
            found += (w in glove)
    return found / total if total else 0

from torch.utils.data import DataLoader, TensorDataset

device = "cuda" if torch.cuda.is_available() else "cpu"
print(f"Training on: {device}")

# ── Convert to tensors ─────────────────────────────────────────────────────────
X_tr = torch.tensor(X_train_emb, dtype=torch.float32)
y_tr = torch.tensor(y_train_rt,  dtype=torch.long)
X_te = torch.tensor(X_test_emb,  dtype=torch.float32)
y_te = torch.tensor(y_test_rt,   dtype=torch.long)

train_loader = DataLoader(TensorDataset(X_tr, y_tr),
                          batch_size=64, shuffle=True)

# ── Instantiate and train MLP ─────────────────────────────────────────────────
mlp = MLPClassifier(input_dim=100, hidden_dim=128, num_classes=2).to(device)
optimizer = torch.optim.Adam(mlp.parameters(), lr=1e-3)
criterion = torch.nn.CrossEntropyLoss()

for epoch in range(10):
    mlp.train()
    total_loss = 0
    for xb, yb in train_loader:
        xb, yb = xb.to(device), yb.to(device)
        optimizer.zero_grad()
        loss = criterion(mlp(xb), yb)
        loss.backward()
        optimizer.step()
        total_loss += loss.item()
    if (epoch + 1) % 2 == 0:
        print(f"  Epoch {epoch+1:2d}  loss={total_loss/len(train_loader):.4f}")

# ── Evaluate ───────────────────────────────────────────────────────────────────
mlp.eval()
with torch.no_grad():
    logits = mlp(X_te.to(device))
    y_pred_mlp = logits.argmax(dim=-1).cpu().numpy()

5.3.3.7.3. GloVe + 1D CNN (Yelp)#

While mean pooling discards all word order information, a 1D Convolutional Neural Network (CNN) applies filters of varying widths over the sequence of word embeddings, capturing local n-gram patterns:

GloVe embedding matrix: (seq_len, 100)
  ↓
Conv1D(filters=128, kernel_size=3) → captures trigram patterns
Conv1D(filters=128, kernel_size=4) → captures 4-gram patterns
Conv1D(filters=128, kernel_size=5) → captures 5-gram patterns
  ↓
MaxPool over each filter → (3 × 128,) = (384,) fixed-length
  ↓
Fully connected → 2 output logits

The CNN learns which short phrase patterns are predictive of each class — for example, a filter that activates strongly on “not worth” (negative) or “highly recommend” (positive). This captures phrase-level patterns that pure mean pooling misses.

5.3.3.7.4. Advantages Over Generation 1#

Dense embedding methods handle vocabulary generalization — if the model has never seen “stupendous” in the training data but knows “outstanding” and “amazing” are nearby in embedding space, it can still classify the review correctly. TF-IDF representations treat each unseen word as zero, discarding potentially useful evidence.

5.3.3.8. Generation 3: Fine-Tuned Transformers#

5.3.3.8.1. RoBERTa Fine-Tuning#

Fine-tuning a pre-trained transformer involves:

  1. Loading a model pre-trained on large corpora (roberta-base []: 125M parameters, trained on 160GB of text).

  2. Replacing the classification head with a task-specific linear layer.

  3. Training the entire model end-to-end on labeled data with a small learning rate.

import torch.nn.functional as F
from torch.nn.utils.rnn import pad_sequence

# ── Tokenize and pad text sequences ───────────────────────────────────────────
MAX_LEN = 128


def texts_to_padded_tensor(texts, glove, dim=100, max_len=MAX_LEN):
    seqs = []
    for text in texts:
        vecs = [torch.tensor(glove[w], dtype=torch.float32)
                for w in text.lower().split()[:max_len] if w in glove]
        if vecs:
            seqs.append(torch.stack(vecs))       # (T, 100)
        else:
            seqs.append(torch.zeros(1, dim))
    # Pad to max_len
    padded = torch.zeros(len(seqs), max_len, dim)
    for i, s in enumerate(seqs):
        l = min(len(s), max_len)
        padded[i, :l] = s[:l]
    return padded                                # (N, max_len, 100)

# ── 1D CNN Classifier ─────────────────────────────────────────────────────────


class CNNClassifier(torch.nn.Module):
    def __init__(self, embed_dim=100, num_filters=128,
                 kernel_sizes=(3, 4, 5), num_classes=2, dropout=0.3):
        super().__init__()
        self.convs = torch.nn.ModuleList([
            torch.nn.Conv1d(embed_dim, num_filters, ks)
            for ks in kernel_sizes
        ])
        self.dropout = torch.nn.Dropout(dropout)
        self.fc = torch.nn.Linear(num_filters * len(kernel_sizes), num_classes)

    def forward(self, x):
        # x: (batch, seq_len, embed_dim) → (batch, embed_dim, seq_len) for Conv1d
        x = x.permute(0, 2, 1)
        pooled = [F.relu(conv(x)).max(dim=-1).values for conv in self.convs]
        out = torch.cat(pooled, dim=-1)   # (batch, num_filters * len(kernels))
        return self.fc(self.dropout(out))

    TensorDataset(X_tr_yelp_seq, y_tr_yelp_t), batch_size=64, shuffle=True)

for epoch in range(5):

    total_loss = 0
    for xb, yb in train_loader_cnn:
        xb, yb = xb.to(device), yb.to(device)

        loss = criterion(cnn(xb), yb)
        loss.backward()
        opt_cnn.step()
        total_loss += loss.item()
    print(f"  Epoch {epoch+1}  loss={total_loss/len(train_loader_cnn):.4f}")

# ── Evaluate CNN ───────────────────────────────────────────────────────────────

5.3.3.9. Generation 3: Fine-Tuned Transformers#

5.3.3.9.1. RoBERTa Fine-Tuning#

Fine-tuning a pre-trained transformer involves loading a model whose weights already encode broad linguistic knowledge, attaching a task-specific classification head, and continuing gradient-based optimization on the labeled training set. The key difference from training from scratch is that the model begins with high-quality representations rather than random weights, requiring far less labeled data to reach strong performance.

RoBERTa [Zhuang et al., 2021] (roberta-base, 125M parameters) was pre-trained on 160 GB of English text using a masked language modeling objective. When fine-tuned for classification, a linear layer is placed on top of the [CLS] token [Devlin et al., 2019] representation — a single vector that the transformer has learned to aggregate sentence-level information into.

Three practical considerations govern fine-tuning:

  1. Learning rate. A small learning rate (\(2 \times 10^{-5}\)) ensures that weight updates are small relative to the pre-trained values, preserving the linguistic knowledge acquired during pre-training.

  2. Weight decay. An L2 penalty on the weights (0.01), applied via the decoupled formulation used by the AdamW optimizer [Loshchilov and Hutter, 2019], acts as a regularizer that prevents the model from overfitting to the relatively small fine-tuning dataset.

  3. Batch size and epochs. Fine-tuning typically converges in 2–4 epochs; more epochs risk overfitting on small datasets.

Key Concept — Fine-Tuning and Catastrophic Forgetting

Fine-tuning adapts a pre-trained model to a downstream task by continuing gradient-based optimization on labeled task data. The fine-tuning objective for binary classification is cross-entropy loss:

(5.37)#\[\begin{equation} \mathcal{L} = -\frac{1}{N} \sum_{i=1}^{N} \left[ y_i \log \hat{p}_i + (1 - y_i) \log (1 - \hat{p}_i) \right] \end{equation}\]

where \(y_i \in \{0, 1\}\) is the true label and \(\hat{p}_i\) is the model’s predicted probability for class 1.

Catastrophic forgetting occurs when a large learning rate causes the fine-tuned weights to drift far from their pre-trained values, erasing the general language knowledge encoded during pre-training. Using a small learning rate (\(2 \times 10^{-5}\)) and weight decay [Loshchilov and Hutter, 2019] constrains the update magnitude, preserving the pre-trained representations while still adapting to the task.

    def __init__(self, texts, labels, tokenizer, max_length=128):

The critical difference from Generations 1 and 2: RoBERTa produces contextualized representations — the same word gets a different vector depending on its surrounding context. “The plot is not without its charms” receives a representation that captures the nuanced sentiment, not just the individual word meanings.

Fine-tuned RoBERTa typically achieves ~87–91% accuracy on Rotten Tomatoes and ~93–95% on Yelp — significantly above classical and embedding-based approaches.

Accuracy alone can be misleading when the costs of different error types are asymmetric, or when the class distribution is imbalanced. A classifier that always predicts the majority class on a 90/10 dataset achieves 90% accuracy while being entirely uninformative. Precision, recall, and F1-score decompose the error structure and expose this failure mode.

The confusion matrix is the foundational accounting tool from which all these metrics are derived. It records, for each combination of true and predicted class, how many examples fall into that cell. From these four counts — true positives, false positives, true negatives, and false negatives — all standard classification metrics follow directly.

Key Definition — Confusion Matrix Quantities

All classification metrics derive from four counts in the confusion matrix:

Predicted Positive

Predicted Negative

Actually Positive

True Positive (TP)

False Negative (FN)

Actually Negative

False Positive (FP)

True Negative (TN)

From these, the primary per-class metrics are:

(5.38)#\[\begin{equation} \text{Precision} = \frac{\text{TP}}{\text{TP} + \text{FP}}, \qquad \text{Recall} = \frac{\text{TP}}{\text{TP} + \text{FN}}, \qquad \text{F1} = \frac{2 \cdot \text{Precision} \cdot \text{Recall}}{\text{Precision} + \text{Recall}} \end{equation}\]

Precision answers: of all items predicted positive, how many actually were?
Recall answers: of all items that actually were positive, how many did we find?

5.3.3.10. Robustness Evaluation#

Aggregate accuracy on a clean test set reveals best-case behavior. Robustness evaluation probes three challenging conditions:

5.3.3.10.1. 1. Negation and Sarcasm#

Negation:  "The film is not without its charms." → Positive (despite "not")
Sarcasm:   "An absolute masterpiece... of wasted potential." → Negative

Classical TF-IDF models are particularly vulnerable: they see the word “masterpiece” and assign positive weight to it, missing the sarcastic framing. RoBERTa, with contextualized attention over the full sentence, handles these cases more reliably.

5.3.3.10.2. 2. Mixed and Neutral Sentiment#

Mixed: "The performances are excellent but the screenplay is a disaster."

All approaches struggle with mixed reviews. The correct label depends on which aspect outweighs the other — a judgment that varies by annotator.
