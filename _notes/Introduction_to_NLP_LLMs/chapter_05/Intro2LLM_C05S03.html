5.3.3. Text Classification#

5.3.3.1. Learning objectives#

By the end of this section, you should be able to:

  • Describe the three generations of text classification approaches: classical sparse-feature pipelines, dense embedding networks, and fine-tuned transformers.

  • Apply TF-IDF with SVM or Naive Bayes and explain when these classical approaches are competitive.

  • Implement a GloVe + neural network classifier and explain the role of pooling in converting a sequence of word vectors to a fixed-length representation.

  • Fine-tune a pre-trained RoBERTa model for binary sentiment classification.

  • Interpret a classification report — distinguishing precision, recall, F1-score, and when each matters most.

  • Evaluate robustness under negation, sarcasm, mixed sentiment, and cross-domain transfer.

5.3.3.2. The Text Classification Problem#

Text classification is the task of assigning a label from a predefined set to a piece of text. It encompasses a wide range of applications:

Task

Input

Output

Sentiment analysis

Movie review

Positive / Negative

Topic classification

News article

Politics / Sports / Technology / …

Spam detection

Email

Spam / Not spam

Intent recognition

User utterance

Book_flight / Check_weather / …

Toxicity detection

Social media post

Toxic / Non-toxic

The core challenge is the same across tasks: how do we convert variable-length text into a fixed-length feature vector that a classifier can operate on?

5.3.3.3. Three Generations of Approaches#

Text classification methods have evolved through three distinct generations, each with fundamentally different representations and models:

Generation 1 — Sparse, count-based features:
  TF-IDF → SVM or Logistic Regression
  TF-IDF → Naive Bayes
  Fast to train, highly interpretable, strong on clean balanced data.

Generation 2 — Dense, distributional embeddings:
  Word2Vec / GloVe + mean pooling → MLP classifier
  Word2Vec / GloVe + 1D CNN → CNN classifier
  Captures semantic similarity; handles synonyms; smaller vocabulary footprint.

Generation 3 — Contextualized transformer representations:
  BERT / RoBERTa fine-tuned end-to-end
  The same word receives different representations in different contexts.
  State-of-the-art performance; expensive to train.

The practices in this section implement all three generations on two benchmark datasets:

  • Rotten Tomatoes (8,530 training / 1,066 test): binary sentiment, short critic snippets.

  • Yelp Polarity (560K available, 12,000 used): binary sentiment, longer business reviews.

5.3.3.4. Generation 1: Classical Sparse Pipelines#

5.3.3.4.1. TF-IDF + SVM (Rotten Tomatoes)#

TF-IDF (Term Frequency–Inverse Document Frequency) converts each document to a sparse vector in which each dimension corresponds to a vocabulary term, weighted by how characteristic that term is of the document:

\[\text{TF-IDF}(t, d) = \text{TF}(t, d) \times \log\frac{N}{1 + \text{df}(t)}\]

Support Vector Machines (SVMs) with a linear kernel find the maximum-margin hyperplane separating the two classes in this high-dimensional sparse space.

from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.svm import LinearSVC
from sklearn.pipeline import Pipeline

pipeline_svm = Pipeline([
    ("tfidf", TfidfVectorizer(max_features=20000, ngram_range=(1, 2),
                               sublinear_tf=True)),
    ("clf",   LinearSVC(C=1.0, max_iter=2000)),
])

pipeline_svm.fit(X_train, y_train)
y_pred = pipeline_svm.predict(X_test)

Typical accuracy on Rotten Tomatoes: ~78–82%. The SVM’s advantages are speed (seconds to train), interpretability (the highest-weight n-grams reveal what the model has learned), and competitive performance on clean, balanced datasets.

5.3.3.4.2. TF-IDF + Naive Bayes (Yelp)#

Multinomial Naive Bayes applies Bayes’ theorem with a feature-independence assumption: given the label, each word’s presence is assumed independent of all other words. Despite this unrealistic assumption, it works well on text:

\[P(\text{label} \mid \text{document}) \propto P(\text{label}) \prod_{t \in \text{document}} P(t \mid \text{label})\]

Key Definition — Multinomial Naive Bayes

Multinomial Naive Bayes estimates the probability of each class given a document by applying Bayes’ theorem:

\[P(y \mid \mathbf{x}) \propto P(y) \prod_{t=1}^{T} P(x_t \mid y)\]

where \(y\) is the class label, \(\mathbf{x} = (x_1, \ldots, x_T)\) are the term counts in the document, and:

  • \(P(y)\) is the class prior (proportion of training documents in class \(y\))

  • \(P(x_t \mid y)\) is the smoothed term likelihood, estimated with Laplace (add-\(\alpha\)) smoothing:

\[P(t \mid y) = \frac{\text{count}(t, y) + \alpha}{\sum_{t'} \text{count}(t', y) + \alpha \cdot |V|}\]

The alpha=0.1 parameter in the code below controls this smoothing strength.

from sklearn.naive_bayes import MultinomialNB

pipeline_nb = Pipeline([
    ("tfidf", TfidfVectorizer(max_features=30000, ngram_range=(1, 2))),
    ("clf",   MultinomialNB(alpha=0.1)),
])

Key advantage: Naive Bayes is the fastest approach in the comparison, with training times under a second even on large datasets. It provides well-calibrated probability estimates and is robust to high-dimensional sparse features.

5.3.3.4.3. What Classical Models Learn#

Inspecting the highest-weight features reveals exactly what TF-IDF models learn:

Top positive features (SVM weights):
  "great", "excellent", "wonderful", "outstanding", "brilliant"

Top negative features:
  "boring", "waste", "terrible", "fails", "disappointing"

These are primarily sentiment lexicon features. The model’s intelligence comes from the training labels assigning these weights, not from any semantic understanding of how words combine.

5.3.3.5. Generation 2: Dense Embedding Networks#

5.3.3.5.1. GloVe + Mean-Pooled MLP (Rotten Tomatoes)#

GloVe (Global Vectors) pre-trains word embeddings on a large corpus by factorizing a word co-occurrence matrix. Each word in the vocabulary receives a fixed 100- or 300-dimensional vector.

To classify a document, we:

  1. Look up the GloVe embedding for each word in the document.

  2. Mean pool all word vectors to produce a single fixed-length document vector.

  3. Pass this vector to a multi-layer perceptron (MLP) classifier.

import numpy as np
import torch
import torch.nn as nn

def embed_documents(texts, glove, dim=100):
    embeddings = []
    for text in texts:
        vecs = [glove[w] for w in text.lower().split() if w in glove]
        if vecs:
            embeddings.append(np.mean(vecs, axis=0))
        else:
            embeddings.append(np.zeros(dim))
    return np.array(embeddings)

class MLPClassifier(nn.Module):
    def __init__(self, input_dim=100, hidden_dim=128, num_classes=2):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(input_dim, hidden_dim),
            nn.ReLU(),
            nn.Dropout(0.3),
            nn.Linear(hidden_dim, num_classes),
        )

    def forward(self, x):
        return self.net(x)

Key Definition — Mean Pooling

Mean pooling aggregates a variable-length sequence of word embeddings into a single fixed-length document vector:

\[\mathbf{d} = \frac{1}{|\mathcal{W}|} \sum_{w \in \mathcal{W}} \mathbf{e}_w\]

where \(\mathcal{W}\) is the set of words in the document that appear in the vocabulary and \(\mathbf{e}_w \in \mathbb{R}^d\) is the pre-trained embedding for word \(w\). Mean pooling is order-invariant: shuffling the words in a document yields the same vector. This is both a strength (no sensitivity to minor word-order variations) and a limitation (negation and word order are invisible).

5.3.3.5.2. GloVe + 1D CNN (Yelp)#

While mean pooling discards all word order information, a 1D Convolutional Neural Network (CNN) applies filters of varying widths over the sequence of word embeddings, capturing local n-gram patterns:

GloVe embedding matrix: (seq_len, 100)
  ↓
Conv1D(filters=128, kernel_size=3) → captures trigram patterns
Conv1D(filters=128, kernel_size=4) → captures 4-gram patterns
Conv1D(filters=128, kernel_size=5) → captures 5-gram patterns
  ↓
MaxPool over each filter → (3 × 128,) = (384,) fixed-length
  ↓
Fully connected → 2 output logits

The CNN learns which short phrase patterns are predictive of each class — for example, a filter that activates strongly on “not worth” (negative) or “highly recommend” (positive). This captures phrase-level patterns that pure mean pooling misses.

5.3.3.5.3. Advantages Over Generation 1#

Dense embedding methods handle vocabulary generalization — if the model has never seen “stupendous” in the training data but knows “outstanding” and “amazing” are nearby in embedding space, it can still classify the review correctly. TF-IDF representations treat each unseen word as zero, discarding potentially useful evidence.

5.3.3.6. Generation 3: Fine-Tuned Transformers#

5.3.3.6.1. RoBERTa Fine-Tuning#

Fine-tuning a pre-trained transformer involves:

  1. Loading a model pre-trained on large corpora (roberta-base: 125M parameters, trained on 160GB of text).

  2. Replacing the classification head with a task-specific linear layer.

  3. Training the entire model end-to-end on labeled data with a small learning rate.

Key Concept — Fine-Tuning and Catastrophic Forgetting

Fine-tuning adapts a pre-trained model to a downstream task by continuing gradient-based optimization on labeled task data. The fine-tuning objective for binary classification is cross-entropy loss:

\[\mathcal{L} = -\frac{1}{N} \sum_{i=1}^{N} \left[ y_i \log \hat{p}_i + (1 - y_i) \log (1 - \hat{p}_i) \right]\]

where \(y_i \in \{0, 1\}\) is the true label and \(\hat{p}_i\) is the model’s predicted probability for class 1.

Catastrophic forgetting occurs when a large learning rate causes the fine-tuned weights to drift far from their pre-trained values, erasing the general language knowledge encoded during pre-training. Using a small learning rate (\(2 \times 10^{-5}\)) and weight decay constrains the update magnitude, preserving the pre-trained representations while still adapting to the task.

from transformers import AutoTokenizer, AutoModelForSequenceClassification
from torch.optim import AdamW

model_name = "roberta-base"
tokenizer  = AutoTokenizer.from_pretrained(model_name)
model      = AutoModelForSequenceClassification.from_pretrained(
    model_name, num_labels=2
)

# Fine-tune with small learning rate to avoid catastrophic forgetting
optimizer = AdamW(model.parameters(), lr=2e-5, weight_decay=0.01)

The critical difference from Generations 1 and 2: RoBERTa produces contextualized representations — the same word gets a different vector depending on its surrounding context. “The plot is not without its charms” receives a representation that captures the nuanced sentiment, not just the individual word meanings.

Fine-tuned RoBERTa typically achieves ~87–91% accuracy on Rotten Tomatoes and ~93–95% on Yelp — significantly above classical and embedding-based approaches.

Key Definition — Confusion Matrix Quantities

All classification metrics derive from four counts in the confusion matrix:

Predicted Positive

Predicted Negative

Actually Positive

True Positive (TP)

False Negative (FN)

Actually Negative

False Positive (FP)

True Negative (TN)

From these, the primary per-class metrics are:

\[\text{Precision} = \frac{\text{TP}}{\text{TP} + \text{FP}}, \qquad \text{Recall} = \frac{\text{TP}}{\text{TP} + \text{FN}}, \qquad \text{F1} = \frac{2 \cdot \text{Precision} \cdot \text{Recall}}{\text{Precision} + \text{Recall}}\]

Precision answers: of all items predicted positive, how many actually were?
Recall answers: of all items that actually were positive, how many did we find?

5.3.3.8. Robustness Evaluation#

Aggregate accuracy on a clean test set reveals best-case behavior. Robustness evaluation probes three challenging conditions:

5.3.3.8.1. 1. Negation and Sarcasm#

Negation:  "The film is not without its charms." → Positive (despite "not")
Sarcasm:   "An absolute masterpiece... of wasted potential." → Negative

Classical TF-IDF models are particularly vulnerable: they see the word “masterpiece” and assign positive weight to it, missing the sarcastic framing. RoBERTa, with contextualized attention over the full sentence, handles these cases more reliably.

5.3.3.8.2. 2. Mixed and Neutral Sentiment#

Mixed: "The performances are excellent but the screenplay is a disaster."

All approaches struggle with mixed reviews. The correct label depends on which aspect outweighs the other — a judgment that varies by annotator.

5.3.3.10. Exercises and discussion#

  1. Feature inspection After training a TF-IDF + SVM classifier, extract the 20 features with the highest positive and negative weights. Are the top features words, bigrams, or a mix? Do any features surprise you — words that you would not have expected to be strongly predictive?

  2. Negation stress test Take 10 positive reviews from the test set and prepend “It is not true that:” to each one. How do the three approaches (TF-IDF+SVM, GloVe+MLP, RoBERTa) respond? Does any approach correctly flip the prediction?

  3. Confusion matrix analysis Plot the confusion matrix for RoBERTa on the test set. Identify the categories of false positives (negative reviews predicted as positive) and false negatives. What properties do the misclassified examples share?
