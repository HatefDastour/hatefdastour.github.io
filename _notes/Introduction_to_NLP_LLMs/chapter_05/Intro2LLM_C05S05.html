5.3.5.3. Why Architecture Has Evolved#

The original transformer architecture [Vaswani et al., 2017] established the foundational design: multi-head self-attention, position-wise feed-forward networks, residual connections, and layer normalization. The models introduced in Chapter 4 and used throughout this chapter are not that original design — they incorporate a series of targeted improvements, each solving a specific problem identified through years of large-scale training experiments.

Understanding these improvements has practical value:

  • Knowing why pre-norm was adopted explains why modern LLMs train more stably than early transformers.

  • Knowing how RoPE works explains why some models handle longer contexts than they were trained on.

  • Knowing what grouped-query attention does explains why 70B models can be served efficiently on fewer GPUs.

5.3.5.4. Improvement 1: Pre-Norm with RMSNorm#

5.3.5.4.1. The Original: Post-Norm#

The original transformer applied layer normalization after the residual addition:

Post-norm (original):
  x → sublayer(x) → x + sublayer(x) → LayerNorm(x + sublayer(x))

Post-norm can produce very large or very small gradient magnitudes at the beginning of training, especially in deep networks. This instability required careful learning rate warmup schedules and sometimes caused training to diverge entirely.

5.3.5.4.2. The Solution: Pre-Norm#

Modern LLMs (LLaMA, Mistral, Phi-3, Gemma) apply normalization before each sublayer:

Pre-norm (modern):
  x → LayerNorm(x) → sublayer(LayerNorm(x)) → x + sublayer(LayerNorm(x))

Pre-norm ensures that each sublayer receives normalized input regardless of the scale of the residual stream. This dramatically improves training stability at large scale and allows higher learning rates.

The code cell below verifies this ordering directly on Phi-3-Mini’s first decoder layer using forward hooks, which record the input and output of a module at the moment it executes during a real forward pass. This avoids reimplementing the layer’s internal control flow and guarantees that the observed ordering matches the model’s actual execution.

Show code cell source

Hide code cell source

 
# Register forward hooks on the sublayers of interest so that we can observe,
# in the actual execution order, when each normalization is applied relative
# to attention and the MLP.

captured = {}


def save_output(name):
    def hook(module, module_inputs, module_output):
        out = module_output[0] if isinstance(
            module_output, tuple) else module_output
        captured[name] = out.detach()
    return hook


handles = [
    layer.input_layernorm.register_forward_hook(
        save_output("x_norm (pre-attention RMSNorm)")),
    layer.self_attn.register_forward_hook(
        save_output("attn_out (self-attention output)")),
    layer.post_attention_layernorm.register_forward_hook(
        save_output("x_norm2 (pre-MLP RMSNorm)")),
    layer.mlp.register_forward_hook(
        save_output("mlp_out (SwiGLU MLP output)")),
]

with torch.no_grad():
    # a full forward pass triggers every hook in execution order
    _ = model(**inputs)

for h in handles:
    h.remove()

for name, tensor in captured.items():
    print(f"{name:35s} shape = {tuple(tensor.shape)}")

5.3.5.4.3. RMSNorm Instead of LayerNorm#

Layer normalization normalizes by computing both the mean and variance across features. Root Mean Square Normalization (RMSNorm) [Zhang and Sennrich, 2019] normalizes by the root mean square only, omitting the mean subtraction.

Both normalizations act feature-wise on a \(d\)-dimensional hidden vector and preserve its dimensionality exactly: the input and output of the normalization layer both lie in \(\mathbb{R}^d\). This invariance is what allows the residual stream \(x \leftarrow x + \text{sublayer}(\text{Norm}(x))\) to remain well-defined across every layer of the network — normalization changes the scale of the vector, never its dimensionality.

Key Formulas — LayerNorm vs. RMSNorm

Layer Normalization (original transformer):

(5.44)#\[\begin{equation} \text{LayerNorm}(x) = \frac{x - \mu}{\sigma + \epsilon} \cdot \gamma + \beta \end{equation}\]

where \(\mu = \frac{1}{d}\sum_i x_i\) is the mean, \(\sigma\) is the standard deviation, \(\gamma\) and \(\beta\) are learned scale and shift parameters, and \(\epsilon\) is a small constant for numerical stability.

Root Mean Square Normalization (modern LLMs):

(5.45)#\[\begin{equation} \text{RMSNorm}(x) = \frac{x}{\text{RMS}(x) + \epsilon} \cdot \gamma, \qquad \text{RMS}(x) = \sqrt{\frac{1}{d}\sum_{i=1}^{d} x_i^2} \end{equation}\]

RMSNorm drops the mean subtraction (\(\mu\)) and the bias (\(\beta\)), reducing the computation to a single statistic. Note that \(\gamma\) is still a learned per-feature scale.

Empirically, the mean-centering term in LayerNorm contributes little to model quality in transformer residual streams: re-centering a vector is a translation, and it is the direction of the hidden state — not its offset from the origin — that carries most of the semantic signal used by downstream dot-product operations such as attention and the LM head. RMSNorm is therefore slightly faster (one statistic instead of two) and has been found to match LayerNorm quality on large language modeling tasks. It is used in LLaMA, Mistral, Phi-3, Qwen, and most post-2022 open-source LLMs.

5.3.5.5. Improvement 2: SwiGLU Activation#

5.3.5.5.1. Original FFN: ReLU#

The original transformer feed-forward network (FFN) applied:

(5.46)#\[\begin{equation} \text{FFN}(x) = \max(0, x W_1 + b_1) W_2 + b_2 \end{equation}\]

ReLU sets all negative values to zero, discarding approximately half of the network’s capacity on any given input. GELU (Gaussian Error Linear Unit) [Hendrycks and Gimpel, 2016], used in BERT [Devlin et al., 2019] and GPT-2, introduced a smooth approximation that partially addresses this.

5.3.5.5.2. SwiGLU: Gated Feed-Forward#

Key Formula — SwiGLU

SwiGLU (Swish-Gated Linear Unit) [Shazeer, 2020] replaces the single non-linearity with a learned gating mechanism:

(5.47)#\[\begin{equation} \text{SwiGLU}(x) = \big(\text{SiLU}(x W_{\text{gate}}) \odot (x W_{\text{up}})\big) \, W_{\text{down}} \end{equation}\]

where \(\text{SiLU}(x) = x \cdot \sigma(x)\) (Sigmoid Linear Unit, also called Swish) [Elfwing et al., 2017], and \(\odot\) is element-wise multiplication. The network learns three projection matrices (\(W_{\text{gate}}\), \(W_{\text{up}}\), \(W_{\text{down}}\)) instead of two.

In Phi-3-Mini, the SwiGLU layers look like this:

Show code cell source

Hide code cell source

 
# gate_up_proj projects the hidden state to twice the MLP intermediate size,
# then the result is split into a gate branch and an up branch of equal width.

gate_up = layer.mlp.gate_up_proj(x)                  # (batch, seq_len, 2 * d_ff)
d_ff = gate_up.shape[-1] // 2
gate, up = gate_up.split(d_ff, dim=-1)                # each (batch, seq_len, d_ff)

import torch.nn.functional as F
output = F.silu(gate) * up                            # SwiGLU gating, (batch, seq_len, d_ff)
output = layer.mlp.down_proj(output)                   # back to (batch, seq_len, hidden_size)

print("gate_up_proj output:", tuple(gate_up.shape))
print("gate / up branch:   ", tuple(gate.shape))
print("SwiGLU output:      ", tuple(output.shape))

The gate \(\text{SiLU}(x W_{\text{gate}})\) learns to selectively suppress or amplify the information in the up-projection \(x W_{\text{up}}\). This gating allows the MLP to learn richer, more conditional functions than a pure element-wise activation, consistently outperforming ReLU and GELU on language modeling tasks.

Why the intermediate dimension is 16384 (not 2 × 3072 = 6144): SwiGLU uses the intermediate dimension for two projections (gate + up), so the effective computation width is 8192. To keep the parameter count comparable to a standard FFN with 4× expansion, the joint projection is set to 2 × 8192 = 16384.

5.3.5.6. Grounding: Scaled Dot-Product Attention Revisited#

Before turning to RoPE, grouped-query attention, and flash attention, it is useful to restate the mechanism that all three modify: scaled dot-product self-attention. Each of the following improvements changes how queries, keys, and values are computed, cached, or masked — none of them changes this underlying operation.

Geometric compatibility. For query vector \(q_i \in \mathbb{R}^{d_k}\) and key vector \(k_j \in \mathbb{R}^{d_k}\), the raw compatibility score \(q_i \cdot k_j\) measures the cosine alignment between the two vectors scaled by their norms. Tokens whose learned query and key representations point in similar directions in \(\mathbb{R}^{d_k}\) receive high compatibility scores and, after the softmax, contribute more of their value vector \(v_j\) to the output. Attention is therefore best understood as a differentiable, content-addressed lookup over the sequence, where addressing is performed by directional alignment in the query/key space rather than by fixed position.

The role of \(1/\sqrt{d_k}\). As \(d_k\) grows, the dot product \(q_i \cdot k_j\) of two independent, zero-mean, unit-variance vectors has variance that scales linearly with \(d_k\). Without correction, this pushes the pre-softmax logits into a regime where softmax gradients vanish almost everywhere except at the single largest entry. Dividing by \(\sqrt{d_k}\) rescales the logits so that their variance stays approximately constant regardless of dimensionality, which keeps the softmax in a numerically well-conditioned regime and preserves usable gradient signal during backpropagation.

Causal masking with \(-\infty\). Autoregressive language models must ensure that the representation at position \(i\) is computed only from positions \(j \le i\). This is enforced by adding a mask \(M\) to the score matrix before the softmax, where \(M_{ij} = -\infty\) for \(j > i\) and \(M_{ij} = 0\) otherwise:

(5.48)#\[\begin{equation} \text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}} + M\right) V \end{equation}\]

Because \(\lim_{z \to -\infty} e^{z} = 0\), masked positions receive exactly zero probability mass after the softmax, regardless of their raw compatibility score. This is a hard, exact constraint rather than a soft penalty — it is what allows a single forward pass over an \(n\)-token sequence to simultaneously produce a valid training signal for all \(n\) next-token predictions.

The multi-head efficiency trade-off. Splitting a \(d_{\text{model}}\)-dimensional representation into \(h\) heads of dimension \(d_k = d_{\text{model}} / h\) lets the model attend to different subspaces — syntactic, positional, semantic — in parallel, at no additional cost in total parameters or FLOPs relative to a single wide attention operation. The trade-off is not in compute but in memory during autoregressive decoding: every head requires its own cached key and value tensor, so increasing \(h\) increases the size of the KV cache proportionally. Grouped-query attention, discussed below, is a direct response to this trade-off.

5.3.5.7. Improvement 3: Rotary Position Encoding (RoPE)#

Position encoding was covered in detail in Section 2, but here we situate it in the broader history of positional encoding approaches:

5.3.5.7.1. Historical Progression#

Year

Method

Key property

2017 (original transformer)

Sinusoidal absolute PE

Fixed; does not generalize beyond training length

2018 (BERT [Devlin et al., 2019])

Learned absolute PE

Flexible during training; fixed at inference

2021 (RoPE, Su et al. [Su et al., 2021])

2022+ (YaRN, LongRoPE)

Extended RoPE variants

Extends context to 128K+ tokens

Key Formula — Rotary Position Encoding (RoPE)

RoPE [Su et al., 2021] encodes positional information by rotating query and key vectors before computing attention. For token at position \(m\), the rotation matrix \(R_m\) rotates each 2D sub-block of the query or key vector by angle \(m \theta_i\), where \(\theta_i = 10000^{-2i/d_k}\) for dimension index \(i\).

The attended output is computed as:

(5.49)#\[\begin{equation} \text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{(R_m Q)(R_n K)^T}{\sqrt{d_k}}\right) V \end{equation}\]

The key property: the dot product \((R_m Q) \cdot (R_n K)\) depends only on the relative offset \(m - n\), not on absolute positions \(m\) and \(n\) individually. This is what enables length generalization.

This property follows directly from the geometric compatibility view introduced above: because \(R_m\) and \(R_n\) are rotation matrices, \(R_m^T R_n = R_{n-m}\), so the inner product \((R_m Q) \cdot (R_n K) = Q^T R_m^T R_n K = Q^T R_{n-m} K\) is a function of \(n - m\) alone. Rotation preserves vector norms, so the magnitude of \(Q\) and \(K\) — and therefore the scale of the attention logits requiring the \(1/\sqrt{d_k}\) correction discussed above — is unaffected by RoPE; only the directional compatibility term is modulated by relative position.

This relative encoding generalizes: a model trained on sequences up to 4,096 tokens can be extended to handle 8,192 or 32,768 tokens through RoPE scaling without any retraining.

5.3.5.8. Improvement 4: Weight Tying#

Weight tying [Press and Wolf, 2017] shares the same parameter matrix between the embedding lookup table and the LM head projection. Instead of maintaining two independent matrices, the model reuses one:

Definition — Weight Tying

In a standard language model, the embedding matrix \(E \in \mathbb{R}^{V \times d}\) maps token IDs to \(d\)-dimensional vectors, while the LM head \(W_{\text{head}} \in \mathbb{R}^{d \times V}\) maps the final hidden state back to vocabulary logits. Weight tying sets \(W_{\text{head}} = E^T\), so both layers share the same \(V \times d\) parameter matrix:

(5.50)#\[\begin{equation} W_{\text{head}} = E^T \end{equation}\]

Show code cell source

Hide code cell source

 
# In Phi-3-Mini:
# model.model.embed_tokens.weight  shared with  model.lm_head.weight

assert model.model.embed_tokens.weight.data_ptr() == model.lm_head.weight.data_ptr()
print("Embedding and LM head share the same underlying parameter tensor.")

The mathematical justification:

  • Input embedding: maps token ID → 3,072-dim vector representing the token’s meaning.

  • LM head: maps 3,072-dim hidden state → score for each token (how probable is this token next?).

  • If the hidden state has learned to resemble the embedding of “Tokyo,” then the dot product with the “Tokyo” embedding row should be high — exactly the behavior we want.

Weight tying saves 98.5M parameters (32,064 × 3,072) with no measurable loss in quality. It is used in GPT-2, LLaMA, Phi-3, and most modern LLMs.

5.3.5.9. Improvement 5: Grouped-Query Attention (GQA)#

5.3.5.9.1. The Memory Problem with Multi-Head Attention#

In standard multi-head attention (MHA), every head has its own unique K and V projection matrices. For a 70B-parameter model with 64 attention heads and 8,192-dim hidden state, the KV cache for a single long sequence can consume tens of gigabytes of GPU memory.

Definition — KV Cache

During autoregressive generation, the model computes key (\(K\)) and value (\(V\)) tensors for every previously generated token. These are stored in the KV cache so they do not need to be recomputed at each new step. KV cache size scales as:

(5.51)#\[\begin{equation} \text{KV cache size} = 2 \times n_{\text{layers}} \times n_{\text{heads}} \times d_{\text{head}} \times n_{\text{tokens}} \times \text{bytes per value} \end{equation}\]

where the factor of 2 accounts for both K and V tensors.

5.3.5.9.2. Grouped-Query Attention#

Grouped-Query Attention (GQA) [Ainslie et al., 2023] directly targets the multi-head efficiency trade-off described above: it reduces memory by sharing K and V projections across groups of query heads while keeping separate Q projections. Compute and expressiveness are largely preserved because every query head still has its own projection into the shared subspace; only the K/V cache — not the FLOPs of the score computation — shrinks. GQA generalizes the more aggressive Multi-Query Attention (MQA) [Shazeer, 2019], which shares a single K/V pair across all query heads.

Multi-Head Attention (MHA):
  32 query heads, 32 key heads, 32 value heads
  → 32 independent K and V projections → large KV cache

Multi-Query Attention (MQA):
  32 query heads, 1 key head, 1 value head
  → 1 K and V shared by all queries → 32× smaller KV cache
  → Quality degradation on some tasks

Grouped-Query Attention (GQA):
  32 query heads, 8 key/value groups (4 queries share each K/V)
  → 4× smaller KV cache vs. MHA, with minimal quality loss

GQA with 4 groups of 8 queries:

  Q₁ Q₂ Q₃ Q₄ Q₅ Q₆ Q₇ Q₈  → attend using K₁, V₁
  Q₉ ... Q₁₆                → attend using K₂, V₂
  ...
  Q₂₅ ... Q₃₂               → attend using K₄, V₄

GQA [Ainslie et al., 2023] is used in LLaMA-3, Mistral, Qwen2, and Phi-3.5. For Phi-3-Mini specifically, the qkv_proj projects to 9,216 = 3 × 3,072 dimensions (MHA with 32 heads), but Phi-3.5 and larger Phi-3 variants use GQA. The cell below reads the number of query and key/value heads directly from the loaded configuration.

5.3.5.10. Improvement 6: Flash Attention#

Flash attention is not an architectural change to the model — it is an algorithmic optimization for computing attention that is mathematically identical but dramatically faster and more memory-efficient on GPU hardware.

5.3.5.10.1. The Problem with Standard Attention#

Standard attention materializes the full \(n \times n\) attention weight matrix in GPU memory:

Standard attention memory: O(n²)
  For n = 4,096 tokens: 4096² = 16.7M elements × 4 bytes = 67 MB
  For n = 32,768 tokens: 32768² = 1.07B elements × 4 bytes = 4.3 GB (!)

For long contexts, this memory requirement becomes prohibitive.

5.3.5.10.2. Flash Attention: Tiling#

`````{admonition} Key Idea — Flash Attention [Dao et al., 2022] :class: tip

Flash Attention avoids materializing the full \(n \times n\) attention matrix by processing tiles of \(Q\), \(K\), \(V\) that fit in the GPU’s fast on-chip SRAM (~256 KB on an A100), accumulating the softmax and output incrementally using the online softmax trick:

(5.52)#\[\begin{equation} m_i = \max_j s_{ij}, \qquad \ell_i = \sum_j e^{s_{ij} - m_i}, \qquad o_i = \frac{1}{\ell_i} \sum_j e^{s_{ij} - m_i} V_j \end{equation}\]

where \(s_{ij} = Q_i K_j^T / \sqrt{d_k}\). These accumulators are updated tile by tile without writing the full matrix to the slower HBM (high-bandwidth memory).

Memory: \(O(n)\) instead of \(O(n^2)\) — the \(n \times n\) matrix is never stored.
Speed: 2–4× faster wall-clock on A100 GPU for long sequences.


The running maximum $m_i$ subtracted before exponentiation is the same numerical-stability device that underlies the $1/\sqrt{d_k}$ scaling and any well-implemented softmax: it keeps the exponentiated terms bounded above by 1. Causal masking composes naturally with tiling — a tile lying entirely above the diagonal (all $j > i$) can be skipped outright, since its contribution to $\ell_i$ and $o_i$ is exactly zero under the $-\infty$ mask, which is one further source of flash attention's speedup for autoregressive decoding.

Flash attention enables training and inference on much longer contexts than would otherwise be feasible. LLaMA-3 (128K context), Phi-3.5 (128K context), and Gemini (1M+ context) all depend on flash attention to operate at these context lengths.

    A LLaMA-3.1-8B model has 32 layers, 32 query heads, and 8 key/value heads (GQA). The hidden dimension is 4,096. For a context of 8,192 tokens in bfloat16 (2 bytes per value), calculate the KV cache size using the formula in the KV Cache definition box. Compare this to what it would be with MHA (32 K/V heads). By how much does GQA reduce the KV cache?

  4. Flash attention tile size
    An A100 GPU has ~40 MB of L2 cache and ~256 KB of on-chip SRAM per streaming multiprocessor. A single tile processes a block of \(B_r\) query rows and \(B_c\) key/value columns. If each value is stored in bfloat16 (2 bytes) and the head dimension \(d_k = 128\), what is the maximum \(B_r = B_c\) that fits in 256 KB?

  5. Causal masking verification
    Using layer.self_attn and the tokenized inputs from the setup cell, construct a \(4 \times 4\) causal mask with \(-\infty\) above the diagonal and \(0\) elsewhere, add it to a toy score matrix of your own construction, and confirm numerically that the post-softmax probabilities for masked positions are exactly zero (not merely small).
