5.3.5. Recent Improvements to the Transformer Architecture#

5.3.5.1. Learning objectives#

By the end of this section, you should be able to:

  • Describe six architectural improvements in modern LLMs relative to the original 2017 transformer: pre-norm with RMSNorm, SwiGLU activation, Rotary Position Encoding (RoPE), weight tying, grouped-query attention, and flash attention.

  • Explain the specific problem each innovation was designed to solve.

  • Identify which innovations are present in Phi-3-Mini from its architecture printout.

  • Describe what flash attention is and why it is important for long-context inference.

  • Connect architectural choices to observable differences in model behavior (training stability, context length, generation speed).

5.3.5.2. Why Architecture Has Evolved#

The original transformer architecture (Vaswani et al., 2017) established the foundational design: multi-head self-attention, position-wise feed-forward networks, residual connections, and layer normalization. The models introduced in Chapter 4 and used throughout this chapter are not that original design — they incorporate a series of targeted improvements, each solving a specific problem identified through years of large-scale training experiments.

Understanding these improvements has practical value:

  • Knowing why pre-norm was adopted explains why modern LLMs train more stably than early transformers.

  • Knowing how RoPE works explains why some models handle longer contexts than they were trained on.

  • Knowing what grouped-query attention does explains why 70B models can be served efficiently on fewer GPUs.

5.3.5.3. Improvement 1: Pre-Norm with RMSNorm#

5.3.5.3.1. The Original: Post-Norm#

The original transformer applied layer normalization after the residual addition:

Post-norm (original):
  x → sublayer(x) → x + sublayer(x) → LayerNorm(x + sublayer(x))

Post-norm can produce very large or very small gradient magnitudes at the beginning of training, especially in deep networks. This instability required careful learning rate warmup schedules and sometimes caused training to diverge entirely.

5.3.5.3.2. The Solution: Pre-Norm#

Modern LLMs (LLaMA, Mistral, Phi-3, Gemma) apply normalization before each sublayer:

Pre-norm (modern):
  x → LayerNorm(x) → sublayer(LayerNorm(x)) → x + sublayer(LayerNorm(x))

Pre-norm ensures that each sublayer receives normalized input regardless of the scale of the residual stream. This dramatically improves training stability at large scale and allows higher learning rates.

# In Phi-3-Mini:
# input_layernorm applied BEFORE self-attention
# post_attention_layernorm applied BEFORE MLP

x_norm = layer.input_layernorm(x)          # normalize first
attn_out = layer.self_attn(x_norm, ...)    # then attend
x = x + attn_out                           # residual

x_norm2 = layer.post_attention_layernorm(x)  # normalize again
mlp_out = layer.mlp(x_norm2)              # then MLP
x = x + mlp_out                            # residual

5.3.5.3.3. RMSNorm Instead of LayerNorm#

Layer normalization normalizes by computing both the mean and variance across features. Root Mean Square Normalization (RMSNorm) normalizes by the root mean square only, omitting the mean subtraction.

Key Formulas — LayerNorm vs. RMSNorm

Layer Normalization (original transformer):

\[\text{LayerNorm}(x) = \frac{x - \mu}{\sigma + \epsilon} \cdot \gamma + \beta\]

where \(\mu = \frac{1}{d}\sum_i x_i\) is the mean, \(\sigma\) is the standard deviation, \(\gamma\) and \(\beta\) are learned scale and shift parameters, and \(\epsilon\) is a small constant for numerical stability.

Root Mean Square Normalization (modern LLMs):

\[\text{RMSNorm}(x) = \frac{x}{\text{RMS}(x) + \epsilon} \cdot \gamma \qquad \text{where} \quad \text{RMS}(x) = \sqrt{\frac{1}{d}\sum_{i=1}^{d} x_i^2}\]

RMSNorm drops the mean subtraction (\(\mu\)) and the bias (\(\beta\)), reducing the computation to a single statistic. Note that \(\gamma\) is still a learned per-feature scale.

RMSNorm is slightly faster (one statistic instead of two) and has been found to match LayerNorm quality on large language modeling tasks. It is used in LLaMA, Mistral, Phi-3, Qwen, and most post-2022 open-source LLMs.

5.3.5.4. Improvement 2: SwiGLU Activation#

5.3.5.4.1. Original FFN: ReLU#

The original transformer feed-forward network (FFN) applied:

\[\text{FFN}(x) = \max(0, x W_1 + b_1) W_2 + b_2\]

ReLU sets all negative values to zero, discarding approximately half of the network’s capacity on any given input. GELU (Gaussian Error Linear Unit), used in BERT and GPT-2, introduced a smooth approximation that partially addresses this.

5.3.5.4.2. SwiGLU: Gated Feed-Forward#

Key Formula — SwiGLU

SwiGLU (Swish-Gated Linear Unit) replaces the single non-linearity with a learned gating mechanism:

\[\text{SwiGLU}(x) = \text{SiLU}(x W_{\text{gate}}) \odot (x W_{\text{up}}) \cdot W_{\text{down}}\]

where \(\text{SiLU}(x) = x \cdot \sigma(x)\) (Sigmoid Linear Unit, also called Swish), and \(\odot\) is element-wise multiplication. The network learns three projection matrices (\(W_{\text{gate}}\), \(W_{\text{up}}\), \(W_{\text{down}}\)) instead of two.

In Phi-3-Mini, the SwiGLU layers look like this:

# gate_up_proj: Linear(3072 → 16384)
# Splits output into gate (8192) and up (8192)

gate_up = layer.mlp.gate_up_proj(x)           # → (batch, seq_len, 16384)
gate, up = gate_up.split(8192, dim=-1)         # split in half

# SwiGLU: apply SiLU to gate, multiply by up
import torch.nn.functional as F
output = F.silu(gate) * up                    # → (batch, seq_len, 8192)
output = layer.mlp.down_proj(output)          # → (batch, seq_len, 3072)

The gate \(\text{SiLU}(x W_{\text{gate}})\) learns to selectively suppress or amplify the information in the up-projection \(x W_{\text{up}}\). This gating allows the MLP to learn richer, more conditional functions than a pure element-wise activation, consistently outperforming ReLU and GELU on language modeling tasks.

Why the intermediate dimension is 16384 (not 2 × 3072 = 6144): SwiGLU uses the intermediate dimension for two projections (gate + up), so the effective computation width is 8192. To keep the parameter count comparable to a standard FFN with 4× expansion, the joint projection is set to 2 × 8192 = 16384.

5.3.5.5. Improvement 3: Rotary Position Encoding (RoPE)#

Position encoding was covered in detail in Section 2, but here we situate it in the broader history of positional encoding approaches:

5.3.5.5.1. Historical Progression#

Year

Method

Key property

2017 (original transformer)

Sinusoidal absolute PE

Fixed; does not generalize beyond training length

2018 (BERT)

Learned absolute PE

Flexible during training; fixed at inference

2021 (RoPE, Su et al.)

Rotation in Q/K space

Relative positions; generalizes to longer sequences

2022+ (YaRN, LongRoPE)

Extended RoPE variants

Extends context to 128K+ tokens

Key Formula — Rotary Position Encoding (RoPE)

RoPE encodes positional information by rotating query and key vectors before computing attention. For token at position \(m\), the rotation matrix \(R_m\) rotates each 2D sub-block of the query or key vector by angle \(m \theta_i\), where \(\theta_i = 10000^{-2i/d_k}\) for dimension index \(i\).

The attended output is computed as:

\[\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{(R_m Q)(R_n K)^T}{\sqrt{d_k}}\right) V\]

The key property: the dot product \((R_m Q) \cdot (R_n K)\) depends only on the relative offset \(m - n\), not on absolute positions \(m\) and \(n\) individually. This is what enables length generalization.

This relative encoding generalizes: a model trained on sequences up to 4,096 tokens can be extended to handle 8,192 or 32,768 tokens through RoPE scaling without any retraining.

5.3.5.6. Improvement 4: Weight Tying#

Weight tying shares the same parameter matrix between the embedding lookup table and the LM head projection. Instead of maintaining two independent matrices, the model reuses one:

Definition — Weight Tying

In a standard language model, the embedding matrix \(E \in \mathbb{R}^{V \times d}\) maps token IDs to \(d\)-dimensional vectors, while the LM head \(W_{\text{head}} \in \mathbb{R}^{d \times V}\) maps the final hidden state back to vocabulary logits. Weight tying sets \(W_{\text{head}} = E^T\), so both layers share the same \(V \times d\) parameter matrix.

# In Phi-3-Mini:
# model.model.embed_tokens.weight  shared with  model.lm_head.weight

assert model.model.embed_tokens.weight.data_ptr() == model.lm_head.weight.data_ptr()
# True — same underlying tensor

The mathematical justification:

  • Input embedding: maps token ID → 3,072-dim vector representing the token’s meaning.

  • LM head: maps 3,072-dim hidden state → score for each token (how probable is this token next?).

  • If the hidden state has learned to resemble the embedding of “Tokyo,” then the dot product with the “Tokyo” embedding row should be high — exactly the behavior we want.

Weight tying saves 98.5M parameters (32,064 × 3,072) with no measurable loss in quality. It is used in GPT-2, LLaMA, Phi-3, and most modern LLMs.

5.3.5.7. Improvement 5: Grouped-Query Attention (GQA)#

5.3.5.7.1. The Memory Problem with Multi-Head Attention#

In standard multi-head attention (MHA), every head has its own unique K and V projection matrices. For a 70B-parameter model with 64 attention heads and 8,192-dim hidden state, the KV cache for a single long sequence can consume tens of gigabytes of GPU memory.

Definition — KV Cache

During autoregressive generation, the model computes key (\(K\)) and value (\(V\)) tensors for every previously generated token. These are stored in the KV cache so they do not need to be recomputed at each new step. KV cache size scales as:

\[\text{KV cache size} = 2 \times n_{\text{layers}} \times n_{\text{heads}} \times d_{\text{head}} \times n_{\text{tokens}} \times \text{bytes per value}\]

where the factor of 2 accounts for both K and V tensors.

5.3.5.7.2. Grouped-Query Attention#

Grouped-Query Attention (GQA) reduces memory by sharing K and V projections across groups of query heads while keeping separate Q projections:

Multi-Head Attention (MHA):
  32 query heads, 32 key heads, 32 value heads
  → 32 independent K and V projections → large KV cache

Multi-Query Attention (MQA):
  32 query heads, 1 key head, 1 value head
  → 1 K and V shared by all queries → 32× smaller KV cache
  → Quality degradation on some tasks

Grouped-Query Attention (GQA):
  32 query heads, 8 key/value groups (4 queries share each K/V)
  → 4× smaller KV cache vs. MHA, with minimal quality loss

GQA with 4 groups of 8 queries:

  Q₁ Q₂ Q₃ Q₄ Q₅ Q₆ Q₇ Q₈  → attend using K₁, V₁
  Q₉ ... Q₁₆                → attend using K₂, V₂
  ...
  Q₂₅ ... Q₃₂               → attend using K₄, V₄

GQA is used in LLaMA-3, Mistral, Qwen2, and Phi-3.5. For Phi-3-Mini specifically, the qkv_proj projects to 9,216 = 3 × 3,072 dimensions (MHA with 32 heads), but Phi-3.5 and larger Phi-3 variants use GQA.

5.3.5.8. Improvement 6: Flash Attention#

Flash attention is not an architectural change to the model — it is an algorithmic optimization for computing attention that is mathematically identical but dramatically faster and more memory-efficient on GPU hardware.

5.3.5.8.1. The Problem with Standard Attention#

Standard attention materializes the full \(n \times n\) attention weight matrix in GPU memory:

Standard attention memory: O(n²)
  For n = 4,096 tokens: 4096² = 16.7M elements × 4 bytes = 67 MB
  For n = 32,768 tokens: 32768² = 1.07B elements × 4 bytes = 4.3 GB (!)

For long contexts, this memory requirement becomes prohibitive.

5.3.5.8.2. Flash Attention: Tiling#

Key Idea — Flash Attention (Dao et al., 2022)

Flash Attention avoids materializing the full \(n \times n\) attention matrix by processing tiles of \(Q\), \(K\), \(V\) that fit in the GPU’s fast on-chip SRAM (~256 KB on an A100), accumulating the softmax and output incrementally using the online softmax trick:

\[m_i = \max_j s_{ij}, \qquad \ell_i = \sum_j e^{s_{ij} - m_i}, \qquad o_i = \frac{1}{\ell_i} \sum_j e^{s_{ij} - m_i} V_j\]

where \(s_{ij} = Q_i K_j^T / \sqrt{d_k}\). These accumulators are updated tile by tile without writing the full matrix to the slower HBM (high-bandwidth memory).

Memory: \(O(n)\) instead of \(O(n^2)\) — the \(n \times n\) matrix is never stored.
Speed: 2–4× faster wall-clock on A100 GPU for long sequences.

Flash attention enables training and inference on much longer contexts than would otherwise be feasible. LLaMA-3 (128K context), Phi-3.5 (128K context), and Gemini (1M+ context) all depend on flash attention to operate at these context lengths.

5.3.5.10. Exercises and discussion#

  1. Architecture archaeology
    Load bert-base-uncased (2018) and microsoft/Phi-3-mini-4k-instruct (2024) and compare their module hierarchies using print(model). For each of the six improvements discussed in this section, determine which model uses the original approach and which uses the improved version.

  2. RMSNorm vs. LayerNorm
    Compute the mean, variance, and RMS of a sample hidden state vector. Verify that LayerNorm and RMSNorm produce different normalized outputs. For which type of vector (large mean, small mean, uniform values) does the mean subtraction in LayerNorm matter most?

  3. GQA memory calculation
    A LLaMA-3.1-8B model has 32 layers, 32 query heads, and 8 key/value heads (GQA). The hidden dimension is 4,096. For a context of 8,192 tokens in bfloat16 (2 bytes per value), calculate the KV cache size using the formula in the KV Cache definition box. Compare this to what it would be with MHA (32 K/V heads). By how much does GQA reduce the KV cache?

  4. Flash attention tile size
    An A100 GPU has ~40 MB of L2 cache and ~256 KB of on-chip SRAM per streaming multiprocessor. A single tile processes a block of \(B_r\) query rows and \(B_c\) key/value columns. If each value is stored in bfloat16 (2 bytes) and the head dimension \(d_k = 128\), what is the maximum \(B_r = B_c\) that fits in 256 KB?
