6.3.4.1. Learning objectives#

  • Implement few-shot Chain-of-Thought (CoT) prompting by providing worked-example reasoning traces.

  • Apply zero-shot CoT using the “Let’s think step by step” trigger and explain why it works.

  • Implement self-consistency decoding and explain how majority voting over multiple reasoning paths improves reliability.

  • Apply tree-of-thought prompting for problems that require exploring multiple solution strategies.

6.3.4.2. Why Standard Prompting Fails on Reasoning Tasks#

When asked a multi-step arithmetic, logic, or planning problem directly, LLMs tend to produce answers in a single forward pass — jumping directly from question to conclusion without constructing an intermediate reasoning chain. This produces high error rates on problems requiring more than two or three logical steps.

The root cause lies in how next-token prediction works: the model predicts each token conditioned on the preceding context, but without an explicit reasoning trace, there is no context to guide correct intermediate steps. Each token in the answer is predicted from the question alone, not from a developing chain of reasoning.

Standard prompt:

Q: If a store sells apples for $0.75 each and a customer buys 4 apples with a $5 bill, what is the change?
A: $2.00

(Often incorrect — model skips intermediate calculation steps)

Chain-of-Thought prompt:

Q: If a store sells apples for $0.75 each and a customer buys 4 apples with a $5 bill, what is the change?
A: Let me work through this step by step.
   Cost per apple: $0.75
   Number of apples: 4
   Total cost: 4 × $0.75 = $3.00
   Change: $5.00 - $3.00 = $2.00
   The change is $2.00.

The reasoning trace forces the model to generate each intermediate step as a token in its output context — turning a single-step prediction problem into an iterative calculation where each step is conditioned on all preceding correct steps.

Before we continue, let’s load back the pip from Section 6.3.1

import torch
from transformers import AutoModelForCausalLM, AutoTokenizer, pipeline

model = AutoModelForCausalLM.from_pretrained(
    "microsoft/Phi-3-mini-4k-instruct",
    device_map="cuda",
    dtype="auto",
    trust_remote_code=False,
)
tokenizer = AutoTokenizer.from_pretrained("microsoft/Phi-3-mini-4k-instruct")

pipe = pipeline(
    "text-generation",
    model=model,
    tokenizer=tokenizer,
    max_new_tokens=200,
    return_full_text=False,
    do_sample=True,
    temperature=0.7,
)

6.3.4.3. Chain-of-Thought: Worked Example in the Prompt#

Few-shot CoT [Wei et al., 2022] provides a complete worked example in the assistant role, demonstrating explicit step-by-step reasoning. The model learns to apply the same pattern to subsequent questions.

Show code cell source

Hide code cell source

 
messages = [
    {
        "role": "system",
        "content": "You solve math problems by showing your work step by step.",
    },
    # Worked example
    {
        "role": "user",
        "content": "A train travels 120 km in 2 hours. How far will it travel in 5 hours at the same speed?",
    },
    {
        "role": "assistant",
        "content": (
            "Step 1: Find the speed.\n"
            "Speed = Distance / Time = 120 km / 2 h = 60 km/h\n\n"
            "Step 2: Find the distance for 5 hours.\n"
            "Distance = Speed × Time = 60 km/h × 5 h = 300 km\n\n"
            "The train will travel 300 km in 5 hours."
        ),
    },
    # Target question
    {
        "role": "user",
        "content": (
            "A cyclist rides 45 km in 3 hours. "
            "How long will it take to ride 90 km at the same speed?"
        ),
    },
]
output = pipe(messages, do_sample=False, temperature=None)[0]["generated_text"]
print(output)

 Step 1: Find the speed.
Speed = Distance / Time = 45 km / 3 h = 15 km/h

Step 2: Find the time for 90 km.
Time = Distance / Speed = 90 km / 15 km/h = 6 hours

It will take the cyclist 6 hours to ride 90 km at the same speed.

By placing the worked example as an assistant turn before the target question, the model has a reasoning template in its immediate context and tends to mirror the structure of the demonstration.

6.3.4.4. Zero-Shot Chain-of-Thought#

Providing worked examples is effective, but requires careful example construction. Zero-shot CoT [Kojima et al., 2022] achieves similar benefits by appending a simple trigger phrase to the prompt:

Show code cell source

Hide code cell source

 
messages = [
    {
        "role": "user",
        "content": (
            "A cyclist rides 45 km in 3 hours. "
            "How long will it take to ride 90 km at the same speed?\n\n"
            "Let's think step by step."  # ← trigger phrase
        ),
    },
]

The phrase “Let’s think step by step” was identified by Kojima et al. [Kojima et al., 2022] as a reliably effective trigger for CoT reasoning in instruction-tuned models. It works because instruction-tuning data contains many examples where deliberate reasoning follows this exact phrasing.

Why it works: The trigger phrase shifts the model’s next-token prediction into a mode where it generates structured intermediate steps — because in the training data, this phrase consistently precedes step-by-step explanations.

Other effective triggers:

  • "Let's work through this carefully."

  • "Think through the problem before answering."

  • "Show your reasoning, then give the final answer."

6.3.4.5. Self-Consistency: Majority Voting Over Reasoning Paths#

A single CoT reasoning trace can still be wrong — the model may take an incorrect intermediate step and reach a wrong conclusion confidently. Self-consistency [Wang et al., 2023] addresses this by generating multiple independent reasoning paths and taking the majority answer.

Show code cell source

Hide code cell source

 
import collections


def self_consistent_answer(question: str, n_samples: int = 5) -> str:
    messages = [
        {
            "role": "user",
            "content": f"{question}\n\nLet's think step by step.",
        },
    ]

    answers = []
    for _ in range(n_samples):
        output = pipe(
            messages,
            do_sample=True,
            temperature=0.7,
            max_new_tokens=300,
        )[0]["generated_text"]

        # Extract the final answer (assumes it appears after "the answer is")
        lines = output.lower().split("\n")
        for line in reversed(lines):
            if "answer is" in line or "result is" in line:
                answers.append(line.strip())
                break
        else:
            answers.append(output.strip().split("\n")[-1])

    # Return the most common answer
    return collections.Counter(answers).most_common(1)[0][0]


result = self_consistent_answer(
    "A factory produces 240 units per day. If it operates 5 days a week, "
    "how many units does it produce in 4 weeks?"
)
print(result)

So, the factory produces 4800 units in 4 weeks.

Why self-consistency works:

Single CoT path → one reasoning trace → one answer (possibly wrong)

Self-consistency with n=5:
  Path 1: correct reasoning → correct answer
  Path 2: arithmetic error → wrong answer
  Path 3: correct reasoning → correct answer
  Path 4: different approach → correct answer
  Path 5: correct reasoning → correct answer

Majority vote: correct answer wins 4:1

The insight is that incorrect reasoning paths rarely converge to the same wrong answer — they tend to fail in different ways. Correct paths, by contrast, tend to converge to the same answer regardless of which approach they take.

Practical parameters:

  • n_samples = 5 is sufficient for most arithmetic and logical reasoning tasks.

  • Use temperature=0.5–0.7 — enough diversity to explore different paths, but not so high that outputs become incoherent.

  • Higher \(n\) gives diminishing returns beyond 10 samples on most benchmarks.

6.3.4.6. Tree-of-Thought: Multiple Expert Perspectives#

For problems where the solution space branches significantly — where different initial strategies may lead to radically different approaches — Tree-of-Thought (ToT) [Yao et al., 2023] prompting asks multiple independent reasoning agents to collaborate.

The key insight: real problem-solving often requires exploring multiple strategies, abandoning dead ends, and backtracking. Standard CoT commits to a single linear path. ToT simulates multiple experts approaching the problem from different angles.

Show code cell source

Hide code cell source

 
tot_prompt = """
Imagine three different experts are solving this problem.
Each expert writes one reasoning step, then all experts share their step.
If any expert realizes their step is wrong, they stop.
The process continues until all experts reach the same conclusion.

Problem: {problem}

Expert 1 (Analytical approach):
"""

problem = (
    "I have 10 books to arrange on a shelf. 3 are red, 4 are blue, and 3 are green. "
    "In how many ways can I arrange them so that all red books are together?"
)

messages = [
    {
        "role": "user",
        "content": tot_prompt.format(problem=problem),
    },
]
output = pipe(messages, do_sample=False, max_new_tokens=400)[
    0]["generated_text"]

6.3.4.7. Exercises and discussion#

  1. CoT vs. direct comparison Select 10 multi-step arithmetic problems. Run each with (a) a direct answer prompt, (b) zero-shot CoT (“Let’s think step by step”), and (c) few-shot CoT with one worked example. Count correct answers in each condition. What is the improvement from direct → zero-shot CoT → few-shot CoT?

  2. Self-consistency calibration Implement self-consistency with n = 1, 3, 5, 10 samples on a set of 20 reasoning problems. Plot accuracy as a function of n. At what value of n does improvement plateau? What is the accuracy cost of using n=1 vs. n=5?

  3. Tree-of-thought decomposition Write a tree-of-thought prompt for a planning problem of your choice (e.g., scheduling, route optimization, or a logic puzzle). Does the model generate genuinely different strategies in each expert branch, or do the branches converge quickly? What does this reveal about the model’s diversity of reasoning?
