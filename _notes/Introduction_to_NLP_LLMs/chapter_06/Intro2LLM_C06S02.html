6.3.2. Advanced Prompt Construction#

6.3.2.1. Learning objectives#

By the end of this section, you should be able to:

  • Assemble a multi-ingredient prompt that combines role, context, task, format, constraints, and tone into a coherent instruction.

  • Apply the specificity principle to transform vague prompts into precise, reliable ones.

  • Explain the primacy and recency effects and use them to position key instructions effectively within a prompt.

  • Apply a hallucination mitigation checklist to reduce factual errors in LLM outputs.

  • Implement chain prompting to decompose a complex task into a sequence of focused, reliable steps.

6.3.2.2. Assembling the Seven-Ingredient Prompt#

Simple prompts produce generic, inconsistently formatted, or poorly targeted responses. Combining the seven ingredients addresses each failure mode systematically.

Consider a task: summarize a technical article about Retrieval-Augmented Generation (RAG) [Lewis et al., 2020] for a corporate executive audience. A naive prompt relies entirely on the model’s default assumptions:

Show code cell source

Hide code cell source

 
messages = [{"role": "user", "content": "Summarize this article about RAG."}]

This inevitably produces a dense, academic summary thick with engineering jargon. To make the output actionable, we assemble a fully-engineered prompt using a multi-role chat structure:

Show code cell source

Hide code cell source

 
text = """
Retrieval-Augmented Generation (RAG) is the process of optimizing the output of a
large language model, so it references an authoritative knowledge base outside of its
training data before generating a response. RAG augments LLMs with up-to-date,
external information without retraining the model, improving accuracy, relevance, and
trust. Typical LLMs only use static training data that can become outdated and may
hallucinate or produce inaccurate information.
"""

messages = [
    {
        "role": "system",
        "content": (
            "You are a professional technical writer "          # Role
            "specializing in AI communication for executives."  # Context
        ),
    },
    {
        "role": "user",
        "content": (
            f"Summarize the following article:\n\n{text}\n\n"  # Task + Context
            "Format your response as:\n"                       # Format
            "  Executive Summary: [2-3 sentence overview]\n"
            "  Key Benefits: [3 bullet points]\n"
            "  Business Implication: [1 sentence]\n\n"
            "Use plain language — no jargon. "                 # Constraint
            "Keep the total response under 150 words."         # Constraint
        ),
    },
]

6.3.2.2.1. Strategic Breakdown of the Blueprint#

By explicitly structuring the prompt components, we isolate distinct execution layers. Each added element directly closes off an unwanted generation path:

  • Role & Context Layer: Shifting the persona to an executive technical writer recalibrates the vocabulary distribution away from raw mathematical engineering terms toward high-level business value.

  • Format Structural Envelope: Forcing a exact markdown key layout ensures downstream readability and predictable token budget allocation across sections.

  • Negative & Structural Constraints: Explicitly forbidding jargon and binding word length prevents conversational drift and ensures high data density.

6.3.2.3. Specificity: The Most Important Principle#

The primary driver of poor LLM performance is semantic vagueness. Specific prompts restrict the network’s available vocabulary to an optimal subset; vague prompts leave the generation path wide open to statistical noise.

  • Vague Prompt: "Explain machine learning."

  • Specific Prompt: "Explain the difference between supervised, unsupervised, and reinforcement learning in exactly three sentences — one per type. Use a real-world analogy for each type. Target audience: a high school student with no prior computer science experience."

To systematically convert any vague instruction into a precise directive, evaluate your prompt against the Five Pillars of Specificity:

  1. Task Execution: What exact mathematical or linguistic transformation should the model perform?

  2. Context & Audience: For whom is this text being generated, and what baseline assumptions apply?

  3. Structural Format: What are the exact structural rules, templates, schemas, or markdown restrictions?

  4. Scope Constraints: What boundaries, forbidden terms, or strict length constraints must be maintained?

  5. Granularity Depth: What exact level of technical depth, abstraction, or detail is required?

6.3.2.4. Primacy and Recency Effects#

Large Language Models do not treat all portions of a long prompt with equal weight. Due to attention distribution dynamics, models exhibit pronounced primacy effects (high attention allocation to the beginning of a prompt) and recency effects (high attention allocation to the end of a prompt).

Conversely, tokens placed in the middle of long contexts often suffer from decreased attention weights—a structural vulnerability documented empirically as the “Lost in the Middle” phenomenon [Liu et al., 2024].

6.3.2.4.1. Optimizing Instruction Placement#

This attention architecture dictates where critical components should be placed within a prompt for maximum execution reliability:

▲ PRIMACY POSITION (Highest Attention Weight)
│  └── System Message / Core Behavioral Constraints
│      → Place your absolute rules, safety guardrails, and role definitions here.
│
┼ MIDDLE POSITION (Lowest Attention Weight)
│  └── Background Data / Context Documents / Raw Text Variables
│      → Place your long data payloads, articles to summarize, or source documents here.
│
▼ RECENCY POSITION (High Attention Weight)
   └── Final Execution Call / Formatting Schema Re-enforcement
       → Place your immediate command, final output template, and vital constraints here.

Engineering Rule of Thumb: If an instruction or formatting constraint is critical to your application’s stability, do not hide it in the middle of a data payload. Repeat the instruction in both the Primacy position (System message) and the Recency position (the final lines of the User message).

Show code cell source

Hide code cell source

 
document = """
Apple Inc. is an American multinational technology company headquartered in Cupertino, California. 
It was founded by Steve Jobs, Steve Wozniak, and Ronald Wayne in 1976.
"""

messages = [
    {
        "role": "system",
        "content": "You always respond in JSON format. No prose, no explanations.",  # primacy
    },
    {
        "role": "user",
        "content": (
            f"Extract the key entities from:\n\n{document}\n\n"
            "Return ONLY valid JSON with keys: {people, organizations, locations}."  # recency
        ),
    },
]

6.3.2.5. Hallucination Mitigation#

Large Language Models optimize for linguistic plausibility rather than factual truth. This makes them prone to generating realistic but entirely fabricated assertions—a critical failure mode known as hallucination [Ji et al., 2023]. While prompt engineering cannot change the underlying model weights to eliminate hallucinations entirely, applying a structural checklist shifts the model from generative guesswork to strict document parsing.

Show code cell source

Hide code cell source

 
document = """
Apple Inc. is an American multinational technology company headquartered in Cupertino, California. 
It was founded by Steve Jobs, Steve Wozniak, and Ronald Wayne in 1976.
"""

6.3.2.5.1. The Grounded Extraction Checklist#

To drastically lower hallucination rates during information extraction, build your prompt using three foundational layers:

  1. Explicit Semantic Grounding: Forcefully restrict the model’s search space to the provided context layer, overriding its pre-trained parametric memory.

  2. Citation Mandates: Require the model to surface raw textual evidence directly alongside its findings. This anchors attention weights back to original substring indices.

  3. Uncertainty Escape Hatches: Explicitly authorize the model to declare an absence of facts rather than forcing it to guess the next token.

Show code cell source

Hide code cell source

 
# Grounded extraction prompt leveraging the recency position
messages = [
    {
        "role": "system", 
        "content": "Answer questions strictly from the provided document. If the information is absent, respond with 'Not stated in source.' Do not assume or extrapolate."
    },
    {
        "role": "user",
        "content": (
            f"Document: \n{document}\n\n"
            "Question: What year did Apple release its first laptop computer?\n\n"
            "Answer using only information explicitly written in the document above."
        )
    }
]

output = pipe(messages)
print(output[0]["generated_text"])

 Not stated in source.

6.3.2.5.2. Structural Reinforcement Strategies#

  • Leverage the Recency Effect: Notice how the constraint “Answer using only information explicitly written…” is placed at the very end of the user message. This ensures the rule occupies the final tokens processed before generation begins.

  • Dual-Pass Verification: For production data pipelines, consider utilizing a secondary, independent LLM call acting as an automated evaluator. The second model is tasked solely with verifying whether the primary model’s generation contains any information missing from the raw reference context.

6.3.2.6. Chain Prompting#

Complex, multi-step operations (e.g., analyzing data, summarizing changes, and drafting an email response) routinely fail when compressed into a single, monolithic prompt. The model’s attention weights get spread too thin across competing goals.

Chain prompting [Wu et al., 2022] solves this by decomposing a complex task into a sequential workflow of discrete API execution states. The output text generated by Step \(N\) is programmatically interpolated directly into the prompt template for Step \(N+1\).

Show code cell source

Hide code cell source

 
# Step 1: Generate product name and slogan
step1_messages = [
    {"role": "system", "content": "You are a creative product naming specialist."},
    {"role": "user", "content": (
        "We're launching an AI tool that helps students study more effectively. "
        "Generate a product name and a one-line slogan. "
        "Format: Name: [name] \nSlogan: [slogan]"
    )}
]
step1_output = pipe(step1_messages)[0]["generated_text"]

# Step 2: Generate targeted copy using Step 1's output as runtime context
step2_messages = [
    {"role": "system", "content": "You are a persuasive copywriter."},
    {"role": "user", "content": (
        f"Using this product information: \n{step1_output}\n\n"
        "Write a 3-sentence sales pitch targeting college students."
    )}
]
step2_output = pipe(step2_messages)[0]["generated_text"]

6.3.2.6.1. Architectural Trade-offs: When to Chain#

Breaking prompts apart provides significant clarity benefits but introduces clear operational overhead:

Table 6.6 Your Table Title Go Here#

Task Characteristics

Single-Prompt Approach

Chain Prompting Paradigm

Simple Extraction / Q&A

Optimal: Low latency, lowest token cost.

Overkill: Adds unnecessary processing steps and API lag.

Multi-Stage Transformation

High Failure Rate: Model often misses middle steps or drops formatting rules.

Recommended: Guarantees each sub-task receives full attention.

Strict Output Schemas

Unreliable: Complex instructions degrade schema adherence.

Recommended: Isolates structural formatting rules to the final step.

Debugging & Observability

Difficult: Opaque failure paths make it hard to tell why a model failed.

Excellent: Pinpoints exactly which step in the pipeline broke.

The rule of thumb: if your prompt contains more than two distinct tasks (generate X, then use X to produce Y), split it into a chain.

6.3.2.7. Exercises and discussion#

  1. Ingredient comparison Write three versions of the same prompt: (a) task only, (b) task + format + constraints, (c) task + format + constraints + role + context. Run each version and compare the outputs along three dimensions: relevance, adherence to format, and factual accuracy.

  2. Primacy vs. recency test Write a prompt with conflicting instructions at the beginning and end (e.g., “respond in Spanish” at the start, “respond in French” at the end). Which instruction does the model follow? Swap the positions and repeat. What does this reveal about primacy vs. recency in your model?

  3. Chain prompting decomposition Take a complex task (e.g., “write a cover letter for a data scientist position at a healthcare company, tailored to the job description below”) and implement it as a three-step chain: (1) extract key requirements from the job description, (2) match your skills to those requirements, (3) draft the cover letter from the matched skills. Compare the output quality to a single-prompt attempt.
