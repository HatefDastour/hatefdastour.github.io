6. Prompt Engineering#

With the internal workings of transformer LLMs explored in Chapter 5, we are now ready to master the practitioner’s most important skill: prompt engineering — the systematic design of inputs that reliably elicit high-quality outputs from language models. This chapter covers the complete spectrum from foundational prompting patterns to advanced reasoning and structured generation techniques.

We begin with the foundations of prompt engineering: what prompts are, why small changes in wording produce large changes in output, and how to decompose effective prompts into their constituent ingredients. We then move to advanced prompt construction — building prompts with multiple layers of instruction, context, format specification, and examples. The middle of the chapter addresses two complementary strategies: in-context learning (teaching the model by example, without any weight updates) and reasoning with generative models (eliciting step-by-step reasoning chains that dramatically improve accuracy on complex tasks). The chapter closes with output verification and structured generation — techniques for making LLM outputs reliable enough for programmatic downstream use — and a practical synthesis that ties all techniques together into a coherent selection framework.

6.1. Learning objectives#

By the end of this chapter, you should be able to:

  • Define prompt engineering and explain why it is the primary tool for steering LLM behavior without modifying model weights.

  • Identify the seven core ingredients of a well-engineered prompt and explain the function of each.

  • Construct prompts using the chat template format for instruction-tuned models, with appropriate system instructions.

  • Apply zero-shot, one-shot, and few-shot prompting and select the appropriate strategy for a given task.

  • Explain how Chain-of-Thought (CoT) prompting elicits step-by-step reasoning and improves accuracy on multi-step problems.

  • Implement self-consistency decoding and explain why majority voting over multiple reasoning paths improves reliability.

  • Apply tree-of-thought prompting to problems requiring exploration of multiple solution strategies.

  • Generate reliable structured output (JSON, YAML) using zero-shot, one-shot, and grammar-constrained generation strategies.

  • Use chain prompting to decompose complex tasks into manageable sequential steps.

  • Select the appropriate prompting technique for a given task using a principled decision framework.

6.2. How this chapter fits into the course#

Chapter 4 introduced LLMs as black boxes that accept prompts and generate text. Chapter 5 opened the box and explained the architecture. This chapter returns to the surface — but now with a deep understanding of the machinery underneath.

Prompt engineering connects forward to every advanced topic in the course:

  • Fine-tuning (Chapter 8) can be understood as a way to encode prompting patterns into model weights — making them reliable without explicit prompting.

  • Retrieval-augmented generation (RAG) relies on prompt engineering to integrate retrieved documents into the model’s context effectively.

  • Agentic AI (Chapter 7) uses chain prompting, tool-use instructions, and ReAct reasoning patterns — all built on the foundations of this chapter.

The techniques covered here apply to any instruction-tuned LLM: Phi-3, Llama-3, Mistral, Gemma, and commercial APIs alike.

6.3. Chapter roadmap#

  1. Foundations of Prompt Engineering We introduce prompt engineering as a discipline, explain why LLMs are sensitive to prompt phrasing, and decompose effective prompts into seven core ingredients: role, context, task, format, constraints, examples, and tone.

  2. Advanced Prompt Construction We assemble multi-ingredient prompts for complex tasks, apply prompt engineering principles (specificity, primacy/recency effects, hallucination mitigation), and introduce chain prompting for multi-step tasks.

  3. In-Context Learning: Zero-Shot, One-Shot, and Few-Shot We explain in-context learning — teaching the model by example in the prompt without any weight updates — and demonstrate zero-shot, one-shot, and few-shot strategies with systematic comparisons.

  4. Reasoning with Generative Models We introduce Chain-of-Thought (CoT) prompting, zero-shot CoT (“let’s think step by step”), self-consistency decoding, and tree-of-thought prompting for problems requiring multi-path reasoning.

  5. Output Verification and Structured Generation We address the challenge of generating reliable structured output (JSON, YAML) for programmatic use, progressing from unreliable zero-shot JSON to schema-locked one-shot prompting to grammar-constrained generation with llama-cpp-python.

  6. Prompt Engineering in Practice We synthesize all techniques into a unified selection framework — given a task, which technique (zero-shot, few-shot, CoT, chain prompting, grammar constraints) should you reach for first? We illustrate the framework on a diverse set of applied NLP tasks.
