4. Introduction to Large Language Models#

With the foundations of text classification and representation learning established in the previous chapters, we are now ready to explore Large Language Models (LLMs) — the family of systems that have transformed natural language processing and artificial intelligence. This chapter introduces LLMs from the ground up, covering both the conceptual and practical dimensions needed to work with them effectively.

We begin with the core idea of causal language modeling: how predicting the next token, applied at enormous scale, produces models capable of answering questions, writing code, summarizing documents, and reasoning about complex topics. We then examine the essentials of LLM architecture, prompting strategies, and generation parameters that control output behavior. The chapter continues with a critical look at LLM robustness — how well models maintain output quality when prompts are noisy, rephrased, or adversarially perturbed. The second half of the chapter focuses on tokens and token embeddings, tracing the complete pipeline from raw text to numbers: tokenization, token ID sequences, static embeddings, contextualized representations, and sentence-level embeddings. We close with comparative analyses of tokenization algorithms across multiple models and a case study of how tokenizer choice affects performance in the high-stakes domain of clinical NLP.

In this chapter, you will learn to:

  • Explain what a large language model is, how causal language modeling works, and how LLMs differ from the classification and embedding models of previous chapters.

  • Describe key properties that distinguish different open-source LLMs: parameter count, context length, architecture, and training approach.

  • Load a pre-trained causal language model and tokenizer using Hugging Face transformers, and use the pipeline abstraction for inference.

  • Construct effective prompts using the chat template format for instruction-tuned models.

  • Control text generation behavior using key parameters: max_new_tokens, temperature, top_p, and repetition_penalty.

  • Evaluate LLM robustness across prompt augmentation strategies (paraphrase, typos/slang, instruction-style variation) and interpret quantitative output metrics.

  • Explain what a token is, why subword tokenization is used, and how WordPiece, BPE, and SentencePiece algorithms differ.

  • Trace the complete pipeline from raw text to token IDs, static embeddings, contextualized embeddings, and sentence embeddings.

  • Compare tokenization behavior across multiple models on the same input and explain how differences affect context window utilization and model behavior.

  • Apply tokenization and embedding analysis to clinical text and explain why tokenizer choice is a practical concern in biomedical NLP.

In Chapters 1 and 2, we established how to acquire, preprocess, and vectorize text. In Chapter 3, we trained supervised classifiers on those representations, progressing from Naive Bayes and logistic regression to fine-tuned BERT models. This chapter marks a qualitative shift: rather than training task-specific classifiers, we work with generative models that learn from the single objective of next-token prediction. This shift opens up text generation, question answering, summarization, and zero-shot task performance without labeled data.

These skills connect forward to:

  • Prompt engineering and fine-tuning modules, where we build on the prompting patterns introduced in Section 1 and tune models for specific tasks.

  • Retrieval-augmented generation (RAG), where sentence embeddings introduced in Section 5 power semantic search and document retrieval.

  • Agentic AI settings, where LLMs serve as reasoning engines within tool-using workflows.

The chapter is organized into seven sections, each corresponding to a teaching session and accompanied by Jupyter-based labs and practice notebooks:

  1. Introduction to Language Models. We introduce causal language modeling, contrast LLMs with the classification models of previous chapters, and walk through loading and running Phi-3-Mini-4K-Instruct — the primary model for this chapter.

  2. Essentials of Large Language Models. We examine the key properties that distinguish LLMs in the open-source ecosystem (parameter count, context length, architecture, quantization), cover the chat template format for instruction-tuned models, and explore generation parameters in detail.

  3. LLM Robustness and Model Comparison. We move from running a single model to systematically evaluating and comparing LLMs. We introduce prompt augmentation strategies and quantitative output metrics (length, type-token ratio, readability, n-gram diversity), and apply them to compare models across multiple domains.

  4. Tokens and Tokenization. We explain what a token is, why character-level and word-level tokenization fail at scale, and how subword algorithms (WordPiece, BPE, SentencePiece) solve the out-of-vocabulary problem. We inspect raw token ID sequences, decode them back to text, and identify the role of special tokens.

  5. Token Embeddings and the Text-to-Numbers Pipeline. We trace the full journey from token IDs through static embedding lookup tables to contextualized transformer representations and finally to sentence-level embeddings via pooling. We use DeBERTa for contextualized token embeddings and Sentence Transformers for semantic similarity tasks.

  6. Tokenization Across Models: A Comparative Study. We compare tokenization behavior across multiple model families (BERT, GPT-2, RoBERTa, Phi-3, Flan-T5) on identical inputs, analyzing how different algorithms handle formal academic language, informal slang, emoji, and code/math notation. We connect tokenizer efficiency differences to context window utilization and generation quality.

  7. Clinical NLP: Tokenization, Embeddings, and Applications. We apply the tokenization and embedding analyses of this chapter to clinical and biomedical text — one of the highest-stakes NLP domains. We audit how general-purpose tokenizers handle drug names, clinical abbreviations, and diagnostic terminology, visualize the resulting embedding geometry, and discuss why domain-specific models such as BioBERT and ClinicalBERT are motivated by these tokenization failures.

Throughout the chapter, code examples accompany each section. Each section includes short, focused Python snippets that illustrate key workflows — such as loading causal language models with transformers, constructing chat-format prompts, inspecting token ID sequences, extracting contextualized embeddings with DeBERTa, and computing semantic similarity with sentence-transformers.
