4.1.2. What is a Large Language Model?#

All the models in the preceding chapters — Logistic Regression, CNN, LSTM, BERT — were trained for a specific task: classify this text as positive or negative, predict the emotion label, assign an economic relevance score. Large Language Models (LLMs) take a fundamentally different approach: instead of learning a fixed task, they learn to predict the next token in a sequence, trained on vast amounts of text.

Key Term — Large Language Model (LLM)

A large language model (LLM) is a neural network, typically with billions of parameters, trained on massive text corpora using a self-supervised objective (usually next-token prediction). The term “large” refers both to parameter count and to the scale of training data. LLMs are generative: given a prefix, they extend it by sampling one token at a time from a learned probability distribution over their vocabulary.

This single objective, scaled to billions of parameters and trillions of tokens, produces models that can answer questions, write code, summarize documents, translate languages, and reason about complex topics — all without task-specific fine-tuning.

The key insight is that next-token prediction is a universal objective: to predict what word comes next in a news article, the model must implicitly understand grammar; to predict the next word in a code comment, it must understand programming conventions; to predict the continuation of a mathematical proof, it must internalize logical structure. Scale and data diversity transform a simple prediction task into general intelligence.

4.1.3. Causal Language Modeling: the Core Objective#

Every LLM introduced in this chapter is a causal language model: it reads tokens left-to-right and predicts what comes next.

Key Term — Causal Language Modeling

Causal language modeling (also called autoregressive language modeling) is the training objective where a model learns to predict the next token given all previous tokens. “Causal” means the model can only attend to tokens that have already appeared in the sequence — it cannot look ahead. This constraint is what makes the model generative: at inference time, it extends a given prefix one token at a time.

Training objective:

  Input:  "The University of Missouri was founded in"
  Target: "1839"

  Next:
  Input:  "The University of Missouri was founded in 1839"
  Target: "and"

  ... repeat across trillions of tokens ...

After training, the model has learned:
  - Grammar and syntax (from repetition across many texts)
  - Facts and world knowledge (from Wikipedia, books, web)
  - Reasoning patterns (from code, math, structured text)
  - Conversational style (from dialogue data)

The word “causal” reflects the constraint that the model can only attend to tokens that have already appeared — it cannot look ahead. This is what makes the model generative: given a prefix, it can extend it one token at a time to produce arbitrary-length outputs.

Key Term — Transformer Architecture

The transformer is the neural network architecture underlying virtually all modern language models. It was introduced by [Vaswani, 2017] in the paper “Attention Is All You Need”. Its key innovation is the self-attention mechanism, which allows every token in a sequence to directly attend to every other token (in encoder models) or every preceding token (in decoder/causal models), enabling the model to capture long-range dependencies that earlier recurrent architectures (LSTMs, GRUs) struggled with. When we say a model is “decoder-only,” we mean it uses the causal variant of the transformer — the architecture used by GPT, Llama, Phi, and the other LLMs in this chapter.

The shift from encoder-only models like BERT to decoder-only causal LLMs is both architectural and philosophical. BERT was designed to produce representations of existing text; causal LLMs are designed to generate new text.

4.1.5. Base Models vs. Instruction-Tuned Models#

A critical distinction when working with LLMs in practice:

Base model (e.g., Llama-3.1-8B):
  Trained only on next-token prediction
  Input:  "The capital of France is"
  Output: "Paris, which is also the country's largest city..."
  (continues the text naturally — does NOT follow instructions)

Instruction-tuned model (e.g., Llama-3.1-8B-Instruct):
  Further trained to follow human instructions via RLHF / SFT
  Input:  [{"role": "user", "content": "What is the capital of France?"}]
  Output: "The capital of France is Paris."
  (answers the question directly and conversationally)

Base models are powerful but require careful prompting to elicit useful behavior. Instruction-tuned models — further trained with Supervised Fine-Tuning (SFT) on human-written instruction-response pairs and/or Reinforcement Learning from Human Feedback (RLHF) — have been adapted to respond helpfully to natural language requests. All models used in the labs for this chapter are instruction-tuned.

Key Term — SFT and RLHF

Supervised Fine-Tuning (SFT) [Ouyang et al., 2022] is the process of continuing to train a pre-trained language model on a curated dataset of instruction–response pairs written or approved by humans. The model learns to follow the format and tone of helpful responses.

Reinforcement Learning from Human Feedback (RLHF) [Ouyang et al., 2022] goes further: human annotators rank multiple model outputs for the same prompt, and a separate reward model is trained to predict these preferences. The language model is then updated with reinforcement learning to maximize this learned reward — pushing it toward outputs that humans find more helpful, harmless, and honest.

Together, SFT and RLHF are the primary techniques that transform a raw language model (one that simply continues text) into an instruction-following assistant.

The primary model for this chapter is Phi-3-Mini-4K-Instruct (3.8B parameters, MIT license). Despite its compact size, it was trained on a highly curated “textbook-quality” dataset and matches or exceeds the performance of many 7B models on reasoning and instruction-following benchmarks.

4.1.7. Loading a Language Model with Hugging Face#

The Hugging Face transformers library provides a unified interface for loading, running, and fine-tuning thousands of pre-trained models. The two core classes for causal language models are AutoTokenizer and AutoModelForCausalLM.

Show code cell source

Hide code cell source

 
import torch
from transformers import AutoModelForCausalLM, AutoTokenizer

# Model identifier on Hugging Face Hub
MODEL_NAME = "microsoft/Phi-3-mini-4k-instruct"

# Pick a dtype based on available hardware (uses torch, avoids unused-import hint)
torch_dtype = torch.float16 if torch.cuda.is_available() else torch.float32

# Load tokenizer and model
tokenizer = AutoTokenizer.from_pretrained(MODEL_NAME)
model = AutoModelForCausalLM.from_pretrained(
    MODEL_NAME,
    device_map="auto",
    dtype=torch_dtype,
    trust_remote_code=False
)

# Quick sanity check
print("Parameters  : ~3.8B")
print(f"PyTorch     : {torch.__version__}")
print(f"Data type   : {next(model.parameters()).dtype}")
print(f"Device      : {next(model.parameters()).device}")

4.1.8. Understanding What the Tokenizer Does#

Before generating text, it is instructive to see exactly what the tokenizer does to our input — this will be explored in depth in Sections 4 and 5.

# Sample text to tokenize
sample = "The University of Missouri was founded in 1839."

# Split text into subword tokens
tokens = tokenizer.tokenize(sample)

# Convert tokens to integer IDs (includes special tokens)
ids = tokenizer.encode(sample)

print(f"Vocabulary size  : {tokenizer.vocab_size:,}")
print(f"Model max length : {tokenizer.model_max_length:,} tokens")
print(f"Tokens : {tokens}")
print(f"IDs    : {ids}")
print(f"Count  : {len(tokens)}")

The tokenizer splits the input string into subword units, maps each unit to an integer index (token ID), and prepends/appends special tokens that the model uses to identify the start and end of sequences.

4.1.9. Exercises and discussion#

  1. Causal vs. masked LM BERT uses a masked language modeling objective (predict randomly masked tokens), while LLMs use causal language modeling [Radford et al., 2025] (predict the next token). What practical consequence does this architectural difference have for how each type of model is used at inference time?

  2. Base vs. instruction-tuned Try passing a direct question (e.g., “What causes inflation?”) to both a base model and its instruction-tuned counterpart (if you have access to both). Describe the qualitative difference in the outputs. Why does this difference arise?

  3. Parameter scale trade-offs Compare a 1.6B model (StableLM-2) to a 3.8B model (Phi-3-Mini) on three short prompts of your choice. For each prompt, note differences in response length, accuracy, and fluency. Based on your observations, when would you prefer the smaller model?
