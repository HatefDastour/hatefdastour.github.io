9. Appendix: Prompt Template Library#

9.1. Template 1: Literature Synthesis#

Role: You are a research methodology expert in [your field].

Context: I'm conducting a literature review on [specific topic]. I've identified [N] key papers published between [date range].

Task: Analyze these papers to:
1. Identify [3-5] major themes across the literature
2. Note areas of consensus and disagreement
3. Highlight methodological approaches used
4. Identify research gaps mentioned by multiple papers

Format: 
- Section for each major theme (100-150 words)
- Bullet points for consensus/disagreement
- Numbered list of research gaps
- Cite papers by first author and year

Papers:
[List papers with abstracts]

9.2. Template 2: Research Question Refinement#

I'm exploring [broad topic area] for my [thesis/dissertation/research project].

Initial rough idea: [vague research question]

Help me refine this by thinking step-by-step:

Step 1: Identify elements that are too vague or broad
Step 2: Narrow the scope (specific population, timeframe, variables)
Step 3: Ensure it's researchable with available methods
Step 4: Propose 3 refined research questions with:
   - Clear independent and dependent variables
   - Specified population
   - Appropriate timeframe
   - Feasibility assessment

Constraints:
- [PhD/Master's/Undergrad] level
- [Available timeframe]
- [Available resources]

9.3. Template 3: Code Documentation#

I have the following [Python/R/other] code that [describe what it does]:

[Paste code]

Please:
1. Add clear, concise comments explaining:
   - Purpose of each major section
   - Logic behind key decisions
   - Any assumptions or limitations
2. Suggest improvements for readability
3. Identify potential bugs or edge cases
4. Add docstrings for functions using [Google/NumPy/other] style

Maintain the existing functionality while improving documentation.

9.4. Template 4: Data Analysis Plan#

Research Context:
- Study design: [experimental/observational/survey/etc.]
- Sample: [description]
- Variables: [list with types: continuous/categorical/ordinal]
- Research question: [specific question]

Create a data analysis plan that includes:
1. Descriptive statistics appropriate for each variable type
2. Assumption checks needed before main analysis
3. Appropriate statistical tests with justification
4. Effect size measures to report
5. Visualization strategy
6. Sensitivity analyses if applicable

Format as structured outline with rationale for each choice. Use [APA 7/AMA/other] statistical reporting standards.

9.5. Template 5: Survey Item Generation#

I'm developing a survey to measure [construct] in [population].

Theoretical framework: [brief description or cite key papers]

Requirements:
- [N] items total
- Mix of positively and negatively worded items
- [Likert 5-point/7-point/other] response scale
- Reading level: [grade level]
- Cultural considerations: [if applicable]

Generate items that:
1. Capture different facets of the construct
2. Are clear and unambiguous
3. Avoid double-barreled questions
4. Are appropriate for target population

For each item, briefly explain what aspect of the construct it measures.

10. Glossary of Key Terms#

Chain-of-Thought (CoT): A prompting technique that encourages step-by-step reasoning by asking the model to show intermediate steps.

Context Window: The maximum amount of text (measured in tokens) that an LLM can process in a single interaction. Information outside this window is not “seen” by the model.

Embedding: High-dimensional vector representations of text that capture semantic meaning, allowing the model to understand relationships between words and concepts.

Few-Shot Learning: Providing 2-5 examples of desired input-output behavior to teach the model through pattern recognition.

Hallucination: When an LLM generates false or fabricated information that sounds plausible but is not factually accurate. Especially problematic with citations and technical facts.

In-Context Learning: The ability of LLMs to learn and perform tasks based solely on examples and instructions provided in the prompt, without changing the model’s underlying parameters.

Parameter: Numerical values learned during training that determine the model’s behavior. More parameters generally enable more complex reasoning but require more computational resources.

Prompt Engineering: The practice of designing effective instructions and queries to maximize the quality and relevance of LLM outputs.

Quantization: Reducing the precision of model parameters (e.g., from 32-bit to 8-bit) to decrease model size and computational requirements with minimal performance loss.

Token: The basic unit of text that LLMs process. Can be a word, part of a word, or punctuation. Critical for understanding context limits and costs.

Zero-Shot Learning: Asking the model to perform a task without any examples, relying entirely on its pre-trained knowledge and the instruction provided.
